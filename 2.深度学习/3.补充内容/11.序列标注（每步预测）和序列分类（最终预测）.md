**序列标注：** 是对序列中每个元素（词 / 字）都输出一个预测结果（逐步标注），
**序列分类：** 是对整个序列只输出一个预测结果（整体判断），两者是 RNN / 深度学习处理序列数据最核心的两类任务形态。

| 维度           | 序列标注（每步预测）                     | 序列分类（最终预测）                |
| ------------ | ------------------------------ | ------------------------- |
| **预测粒度**     | 细粒度：每个词 / 字对应一个预测结果            | 粗粒度：整个序列对应一个预测结果          |
| **输出数量**     | 输出数 = 序列长度（比如 “我→爱→学习” 输出 3 个） | 输出数 = 1（不管序列多长，只输出 1 个）   |
| **核心目标**     | 识别序列中每个元素的属性 / 类别              | 识别整个序列的整体属性 / 类别          |
| **典型应用场景**   | 词性标注、命名实体识别（NER）、分词            | 文本情感分析、文本分类（新闻 / 广告）、意图识别 |
| **RNN 输出方式** | 每个时间步都输出结果                     | 仅最后一个时间步输出结果              |
# 序列标注（每步预测）
# 词性标注
##  训练逻辑
- **目标**：给每个词标注词性（我 = 名词，爱 = 动词，学习 = 动词）。
- **输入**：`[我, 爱, 学习]`
- **真实标签**：`[名词, 动词, 动词]`
- **前向传播**：
    
    1. 输入 “我” → 计算隐藏状态 h1​ → 输出 y1​=Uh1​+by​ → 预测 “名词”
    2. 输入 “爱” → 计算隐藏状态 h2​ → 输出 y2​=Uh2​+by​ → 预测 “动词”
    3. 输入 “学习” → 计算隐藏状态 h3​ → 输出 y3​=Uh3​+by​ → 预测 “动词”
    
- **损失计算**：总 Loss = 预测 “我” 的误差 + 预测 “爱” 的误差 + 预测 “学习” 的误差
- **反向传播**：总 Loss 沿时间轴回传（t3→t2→t1），叠加梯度后更新所有权重（W, V, U, b, by​）。
## 预测逻辑
- 输入新句子：`[我, 爱, 编程]`
- 模型逐词处理：
    1. 输入 “我” → h1​ → y1​ → 输出 “名词”
    2. 输入 “爱” → h2​ → y2​ → 输出 “动词”
    3. 输入 “编程” → h3​ → y3​ → 输出 “动词”
- **最终输出**：`[名词, 动词, 动词]`（与输入等长的标签序列）
# 文本生成
## 训练逻辑
 **前向传播**：    
    - t=1：输入 “我” → 计算隐藏状态 h1​ → 输出 y1​=Uh1​+by​ → 预测 “爱”
    - t=2：输入 “爱” → 计算隐藏状态 h2​ → 输出 y2​=Uh2​+by​ → 预测 “学习”
    - t=3：输入 “学习” → 计算隐藏状态 h3​ → 输出 y3​=Uh3​+by​ → 预测 “编程”
**损失计算**：
    总 Loss = 预测 “爱” 的误差 + 预测 “学习” 的误差 + 预测 “编程” 的误差
 **反向传播**：    
    总 Loss 沿时间轴从 t=3 回传到 t=1，叠加所有时间步的梯度，然后更新所有权重（W, V, U, b, by​）。
    
---
### 预测逻辑（生成文本）
当模型训练好后，输入 `[我, 爱, 学习]`，生成过程如下：
1. t=1：输入 “我” → h1​ → y1​ → 采样得到 “爱”
2. t=2：输入 “爱” → h2​ → y2​ → 采样得到 “学习”
3. t=3：输入 “学习” → h3​ → y3​ → 采样得到 “编程”
4. t=4：将上一步预测的 “编程” 作为新输入 → h4​ → y4​ → 采样得到 “Python”
5. 持续迭代，直到生成结束符（如`<EOS>`）或达到最大长度。
**最终生成结果**：`我 爱 学习 编程 Python ...`
### 文本生成和普通序列标注对照表

| 维度          | 文本生成（语言建模）     | 普通序列标注（如词性标注）       |
| ----------- | -------------- | ------------------- |
| **预测目标**    | 预测当前词的**下一个词** | 预测当前词的**属性 / 标签**   |
| **输出数量**    | 输出数 = 输入序列长度   | 输出数 = 输入序列长度        |
| **Loss 计算** | 所有时间步的误差之和     | 所有时间步的误差之和          |
| **预测方式**    | 可自回归迭代，持续生成新的词 | 仅对输入序列进行一次标注，不生成新内容 |
# 序列分类（以情感分析为例）
## 训练逻辑
- **目标**：判断整句话的情感倾向（我→爱→学习 → 积极）。
- **输入**：`[我, 爱, 学习]`
- **真实标签**：`[积极]`
- **前向传播**：
    1. 输入 “我” → 计算隐藏状态 h1​（不输出）
    2. 输入 “爱” → 计算隐藏状态 h2​（不输出）
    3. 输入 “学习” → 计算隐藏状态 h3​ → 输出 y=Uh3​+by​ → 预测 “积极”
- **损失计算**：总 Loss = 预测整句话情感的误差（仅 1 个）
- **反向传播**：Loss 从 h3​ 回传到 h2​、h1​，叠加梯度后更新所有权重。
## 预测逻辑
- 输入新句子：`[我, 爱, 编程]`
- 模型逐词更新隐藏状态，直到最后一步：
    1. 输入 “我” → h1​
    2. 输入 “爱” → h2​
    3. 输入 “编程” → h3​ → y=Uh3​+by​ → 输出 “积极”
- **最终输出**：`[积极]`（仅 1 个全局标签）
