1. 先看**数据是什么结构**（决定模型大类）
2. 再看**要做什么任务**（分类 / 回归 / 生成 / 决策 / 匹配）
3. 最后看**约束条件**（数据量、算力、部署、时序依赖、图关系）
## 1. 全连接神经网络 MLP / DNN
### 适用数据
- 已经是**固定长度向量 / 特征**的结构化数据
- 表格数据（经过特征工程：类别编码、归一化等）
- 简单低维数据，没有明显空间、时序、拓扑关系
### 典型任务
- 表格数据分类 / 回归（用户点击率预测、简单风控评分、房价预测）
- 作为其他模型的**头部输出层**（CNN/RNN/Transformer 最后接 MLP 做分类）
- 特征维度不高、人工特征已经做得比较好的场景
### 什么时候**不选**MLP
- 数据是图像、语音、长文本、时序、图 → 直接淘汰 MLP
- 数据维度极高（如原始像素）→ 参数爆炸、训练不动、极易过拟合
- 有局部相关性、空间 / 时序结构 → MLP 完全不利用结构信息，效果远差于 CNN/RNN/Transformer
### 一句话决策###
**输入是规整表格 / 固定向量，无空间时序关系 → 用 MLP/DNN**

---
## 2. 卷积神经网络 CNN
### 适用数据
- **二维网格结构数据**：图像、图片、视频帧、频谱图、二维矩阵
- 核心假设：**局部相关性、平移不变性**（近处像素关系远大于远处）
### 典型任务
- 图像分类、目标检测、图像分割、人脸识别、医学影像分析
- 视频帧级处理、图像超分、去噪、风格迁移
- 一维卷积可用于简单时序 / 文本（但一般不如 Transformer/LSTM）
### 什么时候优先 CNN，什么时候用 ViT（Transformer 做视觉）
- 数据偏**传统视觉、算力一般、需要轻量部署、实时性高**（自动驾驶、端侧检测）→ 优先 CNN（YOLO、ResNet、MobileNet 等）
- 数据规模大、追求 SOTA 精度、多模态对齐 → 用 ViT 类 Transformer 模型
### 什么时候**不选**CNN
- 纯文本、长时序依赖、图结构、表格数据 → 不是最优
- 没有空间局部结构的向量数据 → 浪费结构先验
### 一句话决策
**图像 / 视频 / 二维网格，要做视觉感知、分割、检测 → 首选 CNN；大数据高精可上 ViT**

## 3. 循环神经网络 RNN / LSTM / GRU
### 适用数据
- **时序 / 序列数据**：文本、语音波形、时间序列（股价、气温、传感器、日志）
- 核心：**当前信息依赖前面信息**，有序列先后依赖关系
### 典型任务
- 短时序预测、语音识别早期架构、简单机器翻译、情感分析
- 对实时性要求高、序列长度中等的场景
### 优缺点与选型边界
- 优点：轻量、天然串行、适合在线流式推理
- 缺点：**不能并行训练**、长序列梯度消失严重，长文本 / 长时序远不如 Transformer
### 什么时候选 RNN 系，什么时候选 Transformer
- 序列**较短**、算力有限、端侧部署、流式实时处理 → LSTM/GRU
- 序列**很长**、要捕捉长距离依赖、数据量大、追求精度 → 直接上 Transformer
### 一句话决策
**短时序 / 序列、轻量、流式 → LSTM/GRU；长序列、高精度、大数据 → Transformer**

## 4. Transformer（当前通用主流）
### 适用数据
- **几乎所有序列 + 可序列化的数据**：文本、语音、图像（分块后）、视频、多模态
- 核心能力：**自注意力、长距离依赖、并行训练、通用架构**
### 典型任务
- NLP 全领域：翻译、摘要、对话、分类、问答、大模型基座
- 语音识别、语音合成 TTS
- 视觉 SOTA：ViT、DETR、图像分类 / 分割
- 多模态：文生图、图文检索、视频理解
### 什么时候**必选**Transformer
- 长文本、长时序、需要全局依赖关系
- 要做通用大模型、多任务、预训练 + 微调
- 希望一套架构适配 NLP/CV/ 多模态，减少架构切换成本
### 约束
- 算力需求高、参数量大、小样本容易过拟合
- 简单小表格 / 小向量场景，杀鸡用牛刀，不如 MLP
### 一句话决策
**长序列、NLP、多模态、大模型、追求 SOTA 精度 → 无脑优先 Transformer**

## 5. 扩散模型 Diffusion
### 适用任务
- **生成任务**：从噪声逐步还原出高质量数据，不是判别（分类 / 检测）
- 适用模态：图像、视频、音频、3D 点云、分子结构
### 典型任务
- 文生图、图生图、图像修复、超分辨率、人脸生成
- 视频生成、语音合成、高质量素材生成
- 需要生成多样性高、画质精细的场景
### 和 GAN 的简单对比
- Diffusion：质量更高、更稳定、训练更友好，但是生成慢
- GAN：生成快，但难训练、模式崩溃
### 什么时候**不选**Diffusion
- 分类、回归、检测、预测、决策控制 → 完全不适用
- 低延迟实时生成场景（Diffusion 通常步较多，速度不如 GAN / 轻量模型）
### 一句话决策
**要造新图像 / 视频 / 音频，追求高质量生成 → Diffusion；判别类任务直接排除**

## 6. 深度强化学习 DRL
### 核心特点
- **不是用静态数据集训练，而是和环境交互、试错、拿奖励学习策略**
- 目标：学习一个**决策序列**，最大化长期累积回报
### 适用场景
- 智能体决策、控制、博弈、优化序列动作
- 围棋 / 棋牌 AI、机器人控制、自动驾驶决策规划、游戏 AI
- 资源调度、推荐系统在线策略优化、股票动态交易策略
### 什么时候**绝对不选**DRL
- 有标注数据集、做静态分类 / 回归 / 识别 / 分割 / 翻译 → 用监督学习，比 DRL 高效一万倍
- 数据可离线标注，环境稳定可复现 → 优先监督 / 自监督，DRL 成本高、训练不稳定
### 一句话决策
**没有标注数据，只有环境和奖励信号，要学连续动作决策 → DRL；有标注静态数据一律不用**

## 7. 图神经网络 GNN
### 适用数据
- **图结构数据**：节点 + 边，有拓扑关系、关联结构
- 社交网络、知识图谱、推荐用户 - 物品二部图、分子化学结构、路网、论文引用网络
### 典型任务
- 节点分类（用户标签预测）、边预测（好友 / 关联推荐）、图分类（分子属性预测）
- 推荐系统、风控关联分析、药物发现、知识图谱推理
### 什么时候选 GNN
- 数据天然是**关系网络**，个体之间的连接信息比自身特征更重要
- 比如：判断用户是否欺诈，不仅看用户特征，还要看他关联的账户、设备、交易关系
### 什么时候不选
- 无拓扑关系的表格、图像、普通时序、纯文本序列 → 没必要强行构图
- 强行构图反而引入噪声，不如 MLP/CNN/Transformer
### 一句话决策
**数据是节点 + 边的图结构、关系信息很重要 → GNN；无拓扑关系直接排除**
