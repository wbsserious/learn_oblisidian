- **矩阵**：是**二维**的数表，严格限定为**2 阶张量**，是张量的一个特例；
- **张量**：是**多维数组 / 几何不变量**的统称，是标量、向量、矩阵的**高阶泛化**，0 阶、1 阶、2 阶、N 阶都属于张量。
张量（通常指 PyTorch/TensorFlow 中的 Tensor）和 NumPy 数组（ndarray）的区别：
**两者在数据结构和数值计算上高度相似（张量可以看作「增强版」NumPy 数组），但核心差异在于「是否支持自动求导」和「是否适配硬件加速（GPU/TPU）」—— 这也是张量专为深度学习设计的关键特性**。

# 随机张量的创建
- torch.arange()
	线性张量，torch.arange(1,10,1)参考range
	参数1：起始值(包含)；参数2：结束值(不包含)；**参数3：步长**
	torch.arange(0,10,2) # [0,2,4,...,8]
- torch.linspace()
	参数1：起始值(包含)；参数2：结束值(包含)；**参数3：元素个数**
	torch.linspace(0, 9, 10)  # [0,2,4,...,8]
**torch.manual_seed() 设置随机种子**
- torch.rand()
	随机取样，指定形状
	torch.rand(3,4)形状(3 * 4 )，数值0-1随机采样
- torch.randn()
	正态分布，制定现状
	数值范围：理论上无界（−∞,+∞），实际绝大部分落在 [−3,3] 之间（约 99.7%），存在正负值
	数值分布：钟形对称分布（围绕 0 左右波动，0 附近数值出现概率最高，离 0 越远概率越低）
	torch.randn(3,4)形状(3 * 4 )，从标准正态分布N(0,1)随机采样
- torch.randint()
	给定范围，给定形状，在数值范围内随机取样
	torch.randint(0, 10, (3, 4, 5))
	使用torch.randint来创建一个随机张量，形状3*4*5, 从[0,10]随机采样

# 张量元素的类型转换
- .to()
	- `.to()` 是 PyTorch 中**统一张量 / 模型的设备和数据类型**的核心方法，支持单独转换、同时转换两种模式。
	- 张量使用 `.to()` 需**接收返回值**（非原地操作），模型使用 `.to()` 无需接收返回值（原地操作）。

	- **`.type()`**：**专用型类型转换方法**，**仅支持数据类型转换**，不支持设备迁移，且仅能用于「张量」，无法用于模型，功能单一。
	- t3 = t2.type(torch.float32)

`*` 和`@`的区别
`*` 是元素级乘法（逐元素相乘），`@` 是矩阵乘法（对应线性代数中的矩阵点积 / 张量收缩）
- **`*`：元素级乘法（Element-wise Multiplication）**
    - 也叫 “哈达玛积（Hadamard Product）”
    - 规则：两个张量**对应位置的元素逐一相乘**，要求张量形状**要么完全相同，要么满足广播机制**
    - 结果张量的形状和输入张量（广播后的形状）完全一致
- **`@`：矩阵乘法（Matrix Multiplication）**
    - 对应线性代数中的矩阵点积，PyTorch 中等价于 `torch.matmul()`
    - 规则：遵循线性代数的矩阵乘法规则 ——**第一个张量的最后一个维度大小，必须等于第二个张量的倒数第二个维度大小**（多维张量遵循 “最后两维为矩阵维度，前面维度为批次维度” 的规则）
    - 结果张量的形状：除了最后两个维度，其余维度与输入张量的批次维度一致，最后两个维度为 `(第一个张量的倒数第二维, 第二个张量的最后一维)`

# 张量的形状操作
- reshape()         不改变张量内容的前提下，改变张量的形状，不改变元素的顺序
- unsqueeze()       在指定的轴/维度上增加一个维度（长度1），升维
- squeeze()         删除所有长度为1的维度，降维
- transpose()       交换维度，只能交换两个维度
- permute()         交换维度，可以交换多个维度
- view()            改变形状，要求是连续张量，张量中元素的顺序没有改变，没有经过transpose或permute等操作
- contiguous()      将非连续张量转换为连续张量
- is_contiguous()   判断张量是否是连续张量
# 张量的拼接
torch.cat()    
	按指定维度拼接张量，不改变维度数量，除了要拼接的维度，其他维度的形状必须相同   (m, n) + (k, n) -> (m+k, n)
torch.stack()  
	在新的维度上拼接张量，增加了一个新的维度，会改变维度数量，所有张量的形状必须相同   (m, n) + (m, n) -> (2, m, n)
**cat 和 stack的区别：**
cat连接**不会**改变原始数组的维度，按照cat中的dim指标针对相应的维度进行拼接除了dim指定的维度形状不一致，剩下的维度都需要一致
stack连接会改变原始数组的维度。
