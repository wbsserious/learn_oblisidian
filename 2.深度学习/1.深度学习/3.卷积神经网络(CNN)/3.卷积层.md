卷积层的作用
- 卷积层（Convolutional Layer）通过卷积操作提取输入数据中的特征（例如图像中的边缘、纹理、形状等）
- 卷积层利用卷积核（滤波器）对输入进行处理，生成特征图（feature map），并且每个卷积层能够提取不同层次的特征，从低级特征（如边缘）到高级特征（如物体的形状）。
**从输入数据（比如图片）中 “提取特征”** ，形成特征图。
**卷积层的核心要素**：
1. 卷积核
2. 步长
3. 填充
# 卷积核（滤波器）（Kernel）
- **形态**：通常是正方形的小矩阵（3×3 最常用，也有 1×1、5×5 等），也可以是矩形（比如 3×5）；
- **属性**：卷积核的数值（权重）不是固定的，会在模型训练过程中**自动学习调整**—— 训练前是随机值，训练后会变成能精准识别某类特征的数值组合；
- **数量**：一个卷积层可以有多个卷积核（对应 `nn.Conv2d` 的 `out_channels` 参数），每个卷积核负责识别一种特征（比如一个识别竖直线、一个识别水平线、一个识别斜角）。
## 卷积计算
卷积运算本质上就是在卷积核和输入数据的局部区域间做点积。
把卷积核从左到右、从上到下滑动（按照步长移动），重复上述计算，最终得到一张完整的 “特征图”—— 这张图里的数值高低，就代表原图对应位置是否包含该卷积核识别的特征。

# 填充（Padding）
没有填充暴露的问题：
- **尺寸丢失**：每卷积一次，图片尺寸就会变小，多次卷积后会变成极小的特征图，甚至消失；
- **边缘信息丢失**：图片边缘的像素只被卷积核计算 1 次，而中心像素会被计算多次，导致边缘特征（比如图片边缘的物体轮廓）被忽略。
经过卷积后的特征图通常比原始图像小很多。为了保持卷积前后的图像尺寸一致，可以在输入图像的边缘添加 padding（填充）。
padding操作在卷积运算前，在图像边界周围添加额外像素，常用 **零象素填充（zero padding）** 。
**核心目的：**
**1.保持特征图尺寸与输入图片一致；**
**2.减少卷积过程中图像边缘信息的损失。**
### 填充的常见类型（PyTorch 中常用）

| 类型               | 含义                             | 适用场景            |
| ---------------- | ------------------------------ | --------------- |
| `padding=0`      | 无填充（Valid Padding）             | 主动想缩小尺寸（如池化前）   |
| `padding=1`      | 补 1 圈 0（3×3 卷积核的 Same Padding） | 保持尺寸不变（最常用）     |
| `padding=2`      | 补 2 圈 0（5×5 卷积核的 Same Padding） | 5×5 卷积核保持尺寸不变   |
| `padding='same'` | 自动计算填充值（部分框架支持）                | 无需手动算 padding 值 |
> 注意：PyTorch 的`nn.Conv2d`不直接支持`padding='same'`，需要手动计算（3×3 核→padding=1，5×5 核→padding=2）。
# 步长（Stride）
stride（步长）是卷积核在图像上滑动的步幅大小，即两次卷积计算之间卷积核在图像中向右（或向下）移动的像素数。步长影响输出特征图的尺寸。
**步长需要保证卷积核计算区域不会超过图片尺寸**
**核心作用**：
1. **控制输出特征图的尺寸**：步长越大，输出尺寸越小（相当于对特征图做 “下采样”）
2. **调整计算效率**：步长越大，卷积计算的次数越少，模型运行越快（但会丢失更多细节）
3. **间接减小参数/计算量**：更大的步长会让输出特征图更小，后续层的参数量和计算量也会随之降低。
