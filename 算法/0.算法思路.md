## 动态规划思路 ##

> 一种自上而下的问题解决思路，每一个问题的本质是按照一定规则的小问题的解决方法的合集。

> 核心[[递归函数]]：
> 1. 找到自上而下的规则。**递归条件**（recursive case）
> 2. 为了提高效率记录重复情况下的解。
> 3. 找全边界情况。**基线条件**（base case）

> **每个递归函数都有两部分：基线条件**（base case）**和递归条件**（recursive case）

## 贪心算法

>**每步都选“当下看起来最好”的那个选项，永不反悔**
>
>1. 把全局最优解拆成一串**局部决策**；
>2. 对每个局部决策，只根据**当前状态**定一个**立即最优**的选择；
>3. 一旦做出选择，**不回溯、不调整、不等待后续信息**；
>4. 整套决策序列拼起来，恰好就是全局最优（或至少是一个可证明的近似）

>官方样例 
``` python
%% 样例思路 %%
def binary_search(list, item):
  low = 0    （以下2行）low和high用于跟踪要在其中查找的列表部分
  high = len(list)—1  

  while low <= high:  ←-------------只要范围没有缩小到只包含一个元素，
    mid = (low + high) / 2  ←-------------就检查中间的元素
    guess = list[mid]
    if guess == item:  ←-------------找到了元素
      return mid
    if guess > item:  ←-------------猜的数字大了
      high = mid - 1
    else:  ←---------------------------猜的数字小了
      low = mid + 1
  return None  ←--------------------没有指定的元素

my_list = [1, 3, 5, 7, 9]  ←------------来测试一下！

print binary_search(my_list, 3) # => 1  ←--------------------别忘了索引从0开始，第二个位置的索引为1
print binary_search(my_list, -1) # => None  ←--------------------在Python中，None表示空，它意味着没有找到指定的元素
```



## 快速排序
> 对分而治算法的进一步延伸
> 默认处理的数据为无序数据，以数组为例，遍历的方式为，从单签处理的数组中挑选数据为标志点，将数组分为小于标志点数字和高于标志点数字分别组成的数组，在对数组进行拼接。
> （这种方法的问题是，由于多次使用数组，占用空间较大）

样例如下：

```python 
def quicksort(array):
  if len(array) < 2:
    return array  #←------基线条件：为空或只包含一个元素的数组是“有序”的
  else:
    pivot = array[0]  #←------递归条件
    less = [i for i in array[1:] if i <= pivot]  #←------由所有小于等于基准值的元素组成的子数组
    greater = [i for i in array[1:] if i > pivot]  #←------由所有大于基准值的元素组成的子数组

    return quicksort(less) + [pivot] + quicksort(greater)

print quicksort([10, 5, 2, 3])
```

优化后的操作为对原数据进行操作。
