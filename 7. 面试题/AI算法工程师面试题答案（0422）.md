## 一、机器学习 & 深度学习核心理论

### Q1. 偏差、方差、噪声的核心概念，以及高偏差/高方差的故障现象与整改方案

**核心概念：**

| 概念           | 定义                 | 数学含义                           |
| ------------ | ------------------ | ------------------------------ |
| 偏差（Bias）     | 模型预测值的期望与真实值之间的差距  | 模型本身表达能力不足，对数据规律学习不够</br>（欠拟合） |
| 方差（Variance） | 模型在不同训练集上预测结果的波动程度 | 对训练数据噪声过度敏感，泛化能力差</br>（过拟合）    |
| 噪声（Noise）    | 数据本身不可消除的随机误差      | 任何模型理论上都无法拟合的天花板下界             |

**泛化误差 = 偏差² + 方差 + 噪声²**（偏差-方差分解）

**高偏差（欠拟合）故障现象：**
- 大模型微调场景：训练集 loss 居高不下，收敛曲线平坦，SFT 后模型仍答非所问
- 小样本分类：训练集、验证集准确率同步偏低（均在 60%~70% 徘徊），无明显过拟合信号

**高偏差工程整改方案：**
1. 增加模型容量：加深网络层数、增大 LoRA rank（从 8 提升到 32/64）
2. 延长训练轮次：增加 epoch，检查 loss 是否仍有下降空间
3. 优化特征工程：引入更多信息量强的特征，丰富 Prompt 上下文
4. 降低正则化强度：适当减小 weight_decay 或 dropout 率

**高方差（过拟合）故障现象：**
- 训练 loss 持续下降趋近于 0，但验证集 loss 反弹上扬（曲线出现剪刀差）
- 小样本分类：训练集准确率 98%，验证集 60%，泛化严重失效
- 大模型微调：模型死记训练样本，对 OOD（分布外）问题完全崩溃

**高方差工程整改方案：**
1. 数据增强：文本回译、同义词替换、EDA 等扩充数据多样性
2. 增强正则化：提升 dropout 率、增大 L2 weight_decay
3. 减小模型容量：降低 LoRA rank，缩减可训练参数量
4. 早停（Early Stopping）：监控验证集 loss，触发阈值立即停训

### Q2. 交叉熵损失、MSE损失、Focal Loss 底层原理与梯度特性

**交叉熵损失（Cross Entropy）：**

$$CE = -\sum_{i} y_i \log(\hat{y}_i)$$

- 适合分类任务，梯度信号强，收敛快
- 二分类简化为：$CE = -[y\log p + (1-y)\log(1-p)]$
- 缺点：对所有样本一视同仁，样本不均衡时被多数类主导

**MSE损失（均方误差）：**

$$MSE = \frac{1}{n}\sum_i (y_i - \hat{y}_i)^2$$

- 适合回归任务，梯度随误差线性变化
- 分类任务中梯度会饱和（sigmoid 输出接近 0/1 时梯度趋近于 0），不推荐用于分类

**Focal Loss（焦点损失）：**

$$FL = -\alpha(1-p_t)^\gamma \log(p_t)$$

- $\gamma$（聚焦因子，通常取 2）：使易分样本权重指数衰减，强制模型关注难分样本
- $\alpha$：平衡正负样本比例的权重因子

**Focal Loss 适配样本不均衡的核心原因：**
- 医疗文本分类（正常:病变 = 100:1）：易分的大量正常样本 $p_t$ 接近 1，$(1-p_t)^\gamma$ 趋近于 0，其损失权重被大幅压缩，梯度更新几乎全部来自少量难分的病变样本
- 故障识别同理，模型被"迫使"专注于稀有的故障类别，有效解决类别不均衡导致的模型退化问题

### Q3. L1、L2、Dropout、早停策略的作用机制与使用边界

| 正则化方式   | 核心机制                                       | 过拟合场景                    | 欠拟合场景            |
| ------- | ------------------------------------------ | ------------------------ | ---------------- |
| L1正则化   | 损失函数加 $\lambda\|\|w\|\|_1$，使权重稀疏化（部分权重归零）  | ✅ 有效，尤其适合高维稀疏特征（NLP词袋模型） | ❌ 不适用，会进一步压缩模型表达 |
| L2正则化   | 损失函数加 $\lambda\|\|w\|\|_2^2$，使权重均匀缩小（权重衰减） | ✅ 有效，深度学习主流选择（AdamW自带L2） | ❌ 不适用            |
| Dropout | 训练时随机屏蔽 p% 的神经元，推理时关闭                      | ✅ 强过拟合时有效（p=0.3~0.5）     | ❌ 欠拟合时关闭 Dropout |
| 早停      | 监控验证集 loss，连续 N 轮不改善则停止训练                  | ✅ 最简单粗暴的过拟合防治            | ❌ 欠拟合时应延长训练，不应早停 |

**工程使用边界总结：**
- 数据量少 + 模型大 → 同时开启 L2 + Dropout + Early Stopping
- 欠拟合 → 关闭所有正则化，先保证模型充分拟合
- 大模型微调：主要依赖 weight_decay（L2）+ 早停，不建议使用 Dropout（影响预训练权重稳定性）

### Q4. 梯度消失、梯度爆炸的本质诱因及规避方案

**本质诱因：**
- **梯度消失**：反向传播链式求导时，多层激活函数（如 Sigmoid/Tanh）的导数 < 1，连乘后趋近于 0，深层参数几乎不更新
- **梯度爆炸**：权重矩阵特征值 > 1，连乘后指数级放大，参数更新步长失控，loss 变为 NaN

**Transformer 架构规避方案：**
1. **Layer Normalization**：每个子层输出后接 LN，将激活值拉回均值 0、方差 1，阻断梯度异常传播
2. **残差连接（Residual Connection）**：$x + \text{SubLayer}(x)$ 形成梯度高速公路，梯度可直接绕过子层回流，消除消失问题
3. **梯度裁剪（Gradient Clipping）**：设置 `max_norm=1.0`，超出阈值则等比缩放，防爆炸
4. **初始化策略**：Xavier/He 初始化确保初始梯度尺度合理

**深度 CNN 规避方案：**
1. **残差块（ResNet）**：跨层直连，解决 > 50 层时的梯度消失
2. **BatchNorm**：每批次归一化，稳定中间层激活分布
3. **使用 ReLU 替代 Sigmoid**：ReLU 正区间梯度恒为 1，无饱和问题

### Q5. BN 与 LN 的归一化维度、适用场景，以及 Transformer 只用 LN 的原因

**核心区别：**

| 对比维度 | Batch Normalization (BN) | Layer Normalization (LN) |
|---------|-------------------------|-------------------------|
| 归一化维度 | 跨 batch 维度，对每个特征维度在 batch 内统计均值/方差 | 跨特征维度，对单个样本的所有特征统计均值/方差 |
| 依赖关系 | 依赖 batch 内其他样本 | 仅依赖当前样本，样本间独立 |
| 适用场景 | CV（图像分类、目标检测），batch size 较大场景 | NLP、序列模型、Transformer，支持 batch size = 1 |
| 推理阶段 | 需维护全局移动均值/方差 | 无需额外统计量，推理行为与训练完全一致 |

**Transformer 只用 LN 的原因：**
1. **序列长度可变**：NLP 输入句子长度不固定，BN 跨样本统计在 batch 内序列对齐上存在严重问题
2. **小 batch 场景**：大模型训练 per-GPU batch size 极小（1~4），BN 统计不稳定，LN 完全不依赖 batch 大小
3. **自回归生成**：推理时逐 token 生成，实际 batch size = 1，BN 的 running stats 与推理行为不匹配
4. **样本独立性**：LN 使每个 token 的表征归一化独立于其他样本，更适合 attention 机制的语义建模

### Q6. SGD、AdamW、RAdam 对比，以及大模型固定选用 AdamW 的理由

| 优化器 | 收敛速度 | 泛化能力 | 显存占用 | 核心特点 |
|--------|---------|---------|---------|---------|
| SGD | 慢，需精调 lr | 强（工业 CV 常用） | 最低（无动量状态） | 无自适应学习率，依赖人工调参 |
| Adam | 快 | 中等，有过拟合倾向 | 中等（存 m/v 两个矩阵） | 自适应学习率，但 L2 实现有缺陷（权重衰减耦合） |
| AdamW | 快 | 强（解耦 weight decay） | 中等 | Adam 的修正版，将 L2 正则与梯度更新解耦，泛化更好 |
| RAdam | 快 | 强 | 中等 | 冷启动阶段自动降低学习率，前期训练更稳定 |

**大模型训练/微调固定选 AdamW 的工业界理由：**
1. **解耦 weight decay**：传统 Adam 的 L2 正则与自适应学习率耦合，导致正则化效果失真；AdamW 将 weight_decay 直接作用于参数而非梯度，正则化行为符合预期
2. **预训练模型适配**：BERT、GPT、LLaMA 等几乎所有预训练基座均以 AdamW 训练，微调继续使用可保持优化器行为一致
3. **工程成熟度高**：Hugging Face Trainer、DeepSpeed、Megatron 均以 AdamW 为默认优化器，配套 scheduler（linear warmup + cosine decay）已形成最佳实践
4. **显存可接受**：相较于模型参数量，AdamW 额外的 m/v 状态（2倍参数量显存）在混合精度训练下可控

### Q7. 过拟合/欠拟合判别指标与三套调优流程

**判别指标：**

| 信号 | 欠拟合 | 过拟合 |
|------|--------|--------|
| 训练 loss | 高且不下降 | 持续下降趋近于 0 |
| 验证 loss | 高，与训练 loss 接近 | 先降后升（与训练 loss 出现剪刀差） |
| 训练准确率 | 低（< 基准线） | 高（接近 100%） |
| 验证准确率 | 低 | 明显低于训练准确率（差距 > 10%） |
| loss 曲线形态 | 平坦、无收敛趋势 | 训练下降、验证反弹 |

**三套落地调优流程：**

**方案一：过拟合快速整改**
```
1. 检查数据量 → 不足则数据增强（回译、同义词替换）
2. 降低模型复杂度 → 减小 LoRA rank / 减少网络层数
3. 开启 Dropout（0.1~0.3）+ 增大 weight_decay（1e-2）
4. 开启 Early Stopping（patience=3）
5. 使用交叉验证评估泛化真实水平
```

**方案二：欠拟合快速整改**
```
1. 检查 loss 是否正常下降 → 排查 lr 过小（尝试提升 10x）
2. 增加模型容量 → 提升 LoRA rank / 增加隐层维度
3. 关闭所有正则化（Dropout=0，weight_decay=0）
4. 增加训练轮次，延长 warmup steps
5. 检查数据标注质量，排除标签噪声
```

**方案三：训练不稳定（loss 震荡/NaN）**
```
1. 开启梯度裁剪（max_norm=1.0）
2. 降低学习率（缩小 10x），增加 warmup steps
3. 检查数据中是否有异常值/零除操作
4. 降低 batch size，使用梯度累积补偿
5. 混合精度训练中将关键操作改为 FP32
```

### Q8. 精确率、召回率、F1、AUC 与业务场景适配

| 指标 | 公式 | 核心含义 | 适配场景 |
|------|------|---------|---------|
| 精确率（Precision） | TP/(TP+FP) | 预测为正的样本中有多少真正为正 | 风控黑名单（误报代价高，宁可漏掉也不能误伤） |
| 召回率（Recall） | TP/(TP+FN) | 真正为正的样本中有多少被找出 | 医疗病灶检测（漏检代价高，宁可多报也不能漏掉） |
| F1 Score | 2·P·R/(P+R) | 精确率与召回率的调和平均，均衡评估 | 舆情分类、通用文本分类（二者均衡重要） |
| AUC-ROC | ROC 曲线下面积 | 分类器对正负样本排序能力，阈值无关 | 信用评分（关注整体排序能力，不同阈值下均表现稳定） |

**业务场景精细匹配：**
- **风控识别**：优先 Precision → 减少误判正常用户，控制误报率
- **医疗病灶检测**：优先 Recall → 不能漏掉任何病变，允许一定误报
- **舆情分类**：优先 F1 → 正负面舆情均需准确，不可偏废
- **推荐排序**：优先 AUC → 关注整体区分能力，而非单一阈值

### Q2. Transformer Encoder/Decoder 结构与落地模型

**Encoder 结构（每层）：**
```
输入 → Multi-Head Self-Attention → Add & Norm → Feed-Forward Network → Add & Norm → 输出
```
- 每个 token 与序列中所有 token 做双向注意力
- 捕获全局上下文语义

**Decoder 结构（每层）：**
```
输入 → Masked Multi-Head Self-Attention → Add & Norm
     → Cross-Attention（与 Encoder 输出交互）→ Add & Norm
     → Feed-Forward Network → Add & Norm → 输出
```
- Masked 自注意力：防止看到未来 token（因果掩码）
- Cross-Attention：从 Encoder 表征中提取与当前生成位置相关的信息

| 架构类型 | 代表模型 | 适配业务 |
|---------|---------|---------|
| 纯 Encoder | BERT、RoBERTa、BGE | 文本分类、NER、语义向量（Embedding）、问答抽取 |
| 纯 Decoder | GPT 系列、LLaMA、DeepSeek、Qwen | 文本生成、对话、代码补全、指令跟随 |
| Encoder-Decoder | T5、BART、mT5 | 翻译、摘要、条件生成、文档改写 |

### Q3. Self-Attention 计算公式与 QKV 作用

**计算公式：**

$$\text{Attention}(Q,K,V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

**QKV 核心作用：**
- **Q（Query）**：当前 token 想要查找什么信息（"我的需求"）
- **K（Key）**：每个 token 提供的索引/标签（"我能提供什么"）
- **V（Value）**：每个 token 实际携带的信息内容（"信息本体"）
- $\sqrt{d_k}$ 缩放因子：防止 $d_k$ 过大时点积结果过大，导致 softmax 梯度消失

**多头注意力（Multi-Head Attention）的不可替代性：**
- 单头注意力只能关注一种语义关系（如主谓关系）
- 多头将 Q/K/V 投影到 $h$ 个子空间，每个头独立捕获不同类型的依赖关系（句法、语义、共指等），然后拼接
- 表达能力远超单头，且参数量基本不变

**位置编码的不可替代性：**
- Self-Attention 本质是集合操作，对 token 顺序不敏感（"猫吃鱼"和"鱼吃猫"编码相同）
- 位置编码（绝对/相对/RoPE）将序列位置信息注入，赋予模型感知词序的能力

### Q4. 绝对位置编码 vs 相对位置编码 vs RoPE，以及开源模型标配 RoPE 的原因

| 类型 | 代表实现 | 核心机制 | 优点 | 缺点 |
|------|---------|---------|------|------|
| 绝对位置编码 | BERT 可学习编码、Sinusoidal | 直接为每个位置学习/计算独立向量 | 简单，实现方便 | 外推能力差，超出训练长度则崩溃 |
| 相对位置编码 | T5 RPE、ALiBi | 编码 token 对之间的相对距离 | 泛化能力强，支持一定外推 | 计算复杂度高，工程实现较繁琐 |
| 旋转位置编码 RoPE | LLaMA、Qwen、DeepSeek | 通过旋转矩阵将绝对位置编码转化为 Q/K 的相对位置信息 | 外推能力强、实现高效、与注意力机制自然融合 | 超长外推仍需额外技术（YaRN） |

**开源模型（通义千问、DeepSeek）标配 RoPE 的原因：**
1. **自然融合注意力**：RoPE 直接作用于 Q/K 旋转，使点积结果天然反映 token 相对位置，而非外加偏置
2. **外推性能优异**：配合 NTK-aware scaling / YaRN 等技术，可将 4K 上下文窗口扩展到 128K+
3. **计算高效**：无需额外的注意力偏置矩阵，对推理速度几乎无额外损耗
4. **社区验证充分**：RoPE 已被 LLaMA、Mistral、Qwen、DeepSeek 等主流大模型验证，工程生态成熟

### Q5. 预训练 → 微调 → SFT → RLHF 完整链路

| 阶段 | 核心目标 | 数据要求 | 算力成本 |
|------|---------|---------|---------|
| 预训练（Pre-training） | 学习通用语言模式、世界知识、语法语义 | 万亿 token 级别无标注文本（Common Crawl、书籍、代码） | 极高（数千 GPU，数月） |
| 有监督微调 SFT | 让模型学会遵循指令格式，从"续写"转变为"对话/问答" | 数万~数百万条高质量 Prompt-Response 对 | 中等（数十 GPU，数天） |
| 对齐 RLHF | 使输出符合人类偏好（有帮助、无害、诚实） | 人工标注偏好对（Response A vs B，哪个更好） | 中高（需训练 Reward Model + PPO 迭代） |
| 推理部署 | 服务化、低延迟推理 | 无需训练数据 | 低~中（推理卡，按并发扩容） |

**RLHF 具体流程：**
```
1. 收集人类偏好数据（对同一 Prompt 的两个回复打分）
2. 训练 Reward Model（RM）学会预测人类偏好分数
3. 用 PPO 算法以 RM 分数为奖励，微调语言模型
4. 结合 KL 散度约束，防止模型过度优化 RM 而偏离原分布
```

### Q6. Prompt Tuning、Prefix Tuning、LoRA、QLoRA 横向对比

| 方案 | 核心原理 | 显存占用 | 微调速度 | 通用性 | 适用场景 |
|------|---------|---------|---------|--------|---------|
| Prompt Tuning | 在输入前添加可学习的软 token，只训练这些 token | 极低 | 快 | 差（只学到特定 Prompt 偏置） | 极低资源场景，单任务适配 |
| Prefix Tuning | 在每层 Attention 的 K/V 前拼接可学习前缀向量 | 低 | 快 | 中等 | 序列生成类任务，如摘要、翻译 |
| LoRA | 在权重矩阵旁添加低秩分解矩阵 $W = W_0 + BA$（$B \in \mathbb{R}^{d \times r}, A \in \mathbb{R}^{r \times k}$），只训练 A/B | 低~中 | 快 | 强（合并后无推理开销） | 工业最主流，通用场景微调首选 |
| QLoRA | LoRA + 4bit NF4 量化基础模型 + double quantization + paged optimizer | 极低 | 较快 | 强 | 单卡 24GB 微调 70B 级模型 |

**rank 参数选择建议：**
- 通用能力轻微调：r=4~8
- 行业专属知识注入：r=16~32
- 复杂推理/代码能力：r=64

### Q7. KV Cache 的核心作用与优化策略

**核心作用：**
- 大模型自回归推理时，每生成一个新 token 需重新计算所有历史 token 的 K/V
- KV Cache 将已计算的 K/V 矩阵缓存在显存中，新 token 只需计算自身的 K/V 并追加，历史部分直接复用
- 效果：推理速度从 $O(n^2)$ 降低到 $O(n)$（n 为已生成长度）

**优化配置策略：**
1. **Multi-Query Attention（MQA）**：多个 Q 头共享同一组 K/V，KV Cache 显存降低至 1/head 数
2. **Grouped Query Attention（GQA）**：MHA 和 MQA 的折中，分组共享 K/V（LLaMA 3、Qwen2 采用）
3. **PagedAttention（vLLM）**：将 KV Cache 分页管理，消除显存碎片，支持动态显存分配
4. **Prefix KV Cache 共享**：相同 System Prompt 的 KV Cache 一次计算全量复用
5. **KV Cache 量化**：将 K/V 量化为 INT8，显存减半，精度损失极小

**轻微副作用：**
- 显存占用随序列长度线性增长，极长序列（32K+）时显存压力大
- 多用户并发时 KV Cache 不可跨请求共享（除 Prefix Cache 外），显存利用率下降

### Q8. 上下文窗口限制根源与有效对话长度拓展方案

**限制根源：**
1. **算力**：Self-Attention 复杂度为 $O(n^2 \cdot d)$，序列越长算力消耗平方增长
2. **显存**：KV Cache 随序列长度线性增长，64K token 时 KV Cache 可占用数十 GB 显存
3. **训练分布**：模型只在固定长度（如 4096）内训练，超出后位置编码失效，性能急剧下降

**低成本拓展有效对话长度方案：**

**方案一：滑动窗口（Sliding Window）**
- 每次只保留最近 $w$ 个 token 的完整注意力，远距离 token 使用稀疏注意力（Longformer、Mistral）
- 适合：流式对话，历史信息重要性递减的场景

**方案二：上下文压缩（Context Compression）**
- 用小模型对历史对话进行摘要，压缩为固定长度的"记忆摘要"注入 System Prompt
- 适合：多轮长对话助手，历史关键信息需保留但篇幅需压缩

**方案三：位置编码外推（RoPE Scaling）**
- NTK-aware scaling、YaRN 等技术调整 RoPE 的旋转频率，将 4K 模型无损外推到 32K~128K
- 适合：文档阅读、长文本理解，需要全局注意力

**方案四：RAG 替代长上下文**
- 将历史信息向量化入库，按需检索相关片段注入，而非全量塞入上下文
- 最低成本，对极长历史信息效果最好

### Q2. BM25 vs 稠密向量检索，以及工业界要求混合检索的原因

| 对比维度 | BM25（稀疏检索） | 稠密向量检索（Dense Retrieval） |
|---------|----------------|-------------------------------|
| 底层原理 | 基于词频/逆文档频率的统计匹配 | Embedding 语义相似度（余弦/内积） |
| 精确词匹配 | ✅ 非常强（专有名词、产品型号、编码） | ❌ 弱（语义近似但词不同则可能漏检） |
| 语义泛化 | ❌ 无法理解同义词、近义表达 | ✅ 非常强（"心脏病"和"冠心病"语义相近） |
| 硬件成本 | 极低（倒排索引，内存友好） | 中高（需 GPU 编码 + 向量存储） |
| 检索速度 | 极快（毫秒级） | 快（近似最近邻搜索） |

**工业界强制要求混合检索的原因：**
- 纯 BM25：遇到同义词、缩写、多语言表达则大量漏检（医疗场景尤甚）
- 纯向量检索：专有名词（产品编号 "X-2307B"、法规条款编号）语义向量近似度低，精确匹配失败
- **混合检索（BM25 + Dense + Reranker）= 全面覆盖**：BM25 保精确词匹配，向量检索保语义泛化，Reranker 做精排，互补缺陷

### Q3. Embedding 检索召回率低的 5 个核心根因与优化方案

| ⑤ 向量库索引类型不合适 | HNSW 参数 ef_search 过小 | 调大 ef_search/nprobe，使用更精确的索引类型（Flat 用于小规模） |

### Q6. 幻觉的三层根治架构

**幻觉定义**：模型生成了看似合理但实际错误或无依据的内容

**三层根治架构：**

**第一层：数据源溯源**
- 为每个检索片段标注来源（文件名、页码、段落ID）
- 生成答案时强制引用源文档，并在前端展示原文对照
- 用户可一键验证答案与原文的一致性

**第二层：检索过滤**
- 设置相似度阈值：低于 0.75 的检索结果直接过滤，不进入 Prompt
- 引入 Reranker 二次精排，过滤低相关度片段
- 检索结果为空时，触发拒答机制："根据现有知识库未找到相关信息"

**第三层：Prompt 约束**
- 在 System Prompt 中明确约束："只根据以下参考资料回答，如果资料中没有明确说明，请回答'根据现有资料无法回答'"
- 开启 Temperature=0（或低温）提高生成确定性
- 事后用 NLI 模型自动检测答案与来源的一致性（Answer Faithfulness 评估）

### Q7. Reranker 工作原理与效果提升原因

**工作原理：**
- 向量检索召回 Top-20~50 候选文档（粗排）
- Reranker（如 BGE-Reranker、cross-encoder）将 Query 与每个候选文档**拼接**输入 BERT 类模型，输出精确相关性分数（精排）
- 取精排 Top-5 进入 LLM 生成

**为什么小体量 Reranker 能大幅提升精度且不显著增加耗时：**
1. **计算量可控**：向量检索已将候选数从百万压缩到 20~50，Reranker 只需对少量候选对做精确打分
2. **Cross-Encoder 的信息优势**：Bi-Encoder（向量检索）将 Query 和文档独立编码，无法感知二者之间的精细交互；Cross-Encoder 将二者拼接，能捕捉词级别的精确匹配和语义对齐，精度远超 Bi-Encoder
3. **轻量模型即可**：BGE-Reranker-base（110M 参数）对 50 个候选打分仅需 100~200ms，相比 LLM 推理耗时（数秒）可忽略不计

### Q8. 多模态 RAG 与纯文本 RAG 的核心架构差异

**核心差异：**

| 维度 | 纯文本 RAG | 多模态 RAG |
|------|-----------|-----------|
| 文档解析 | 文本提取即可 | 需解析图片、表格、公式（OCR / 版式解析） |
| 向量化 | 纯文本 Embedding | 文本+图片多模态 Embedding（CLIP、BGE-M3） |
| 检索 | 文本向量相似度 | 跨模态检索（文本Query → 图片文档） |
| 生成 | 纯文本 LLM | 多模态 LLM（GPT-4V、Qwen-VL） |

**图片/表格类工业文档处理流程：**

```
2. 表格处理：
   - 方案A：将表格转为 Markdown 文本（保留结构信息）→ 文本向量化
   - 方案B：表格截图 → 多模态 Embedding（CLIP）→ 图文联合索引
3. 图片处理：
   - 使用多模态模型（如 LLaVA）生成图片描述文本 → 文本向量化入库
   - 或直接使用 CLIP 图像向量入库，支持图文跨模态检索
4. 检索联动：文本向量 + 图像向量分别检索，按相关度融合后统一送入多模态 LLM
```

## 四、大模型微调 + 部署工程

### Q1. LoRA 微调核心原理与低显存优势

**核心原理：**

$$W = W_0 + \Delta W = W_0 + BA$$

- $W_0 \in \mathbb{R}^{d \times k}$：冻结的预训练权重
- $B \in \mathbb{R}^{d \times r}, A \in \mathbb{R}^{r \times k}$：可训练的低秩矩阵，$r \ll \min(d,k)$
- 初始化：$A$ 使用随机高斯初始化，$B$ 初始化为全零（确保初始 $\Delta W = 0$）

**低显存低成本的原因：**
1. **冻结基础模型**：7B 模型全参数微调需约 112GB 显存（BF16），LoRA 只训练约 0.1%~1% 参数
2. **参数量极小**：r=16 时，单个 Attention 矩阵的 LoRA 参数 = $d \times r + r \times k$（远小于 $d \times k$）
3. **合并无推理开销**：推理前可将 $BA$ 合并回 $W_0$，完全不影响推理速度
4. **优化器状态显存降低**：AdamW 优化器状态为参数量的 2 倍，LoRA 可训练参数少，优化器显存成比例降低

---

### Q2. QLoRA 量化精度选择与低显存微调技巧

**4bit vs 8bit 取舍：**

| 量化精度 | 显存占用 | 精度损失 | 微调效果 | 适用场景 |
|---------|---------|---------|---------|---------|
| INT8 | 基础模型降为约 50% | 极小（< 0.5% 性能损失） | 接近全精度 | 24GB 显卡微调 13B 模型 |
| NF4（4bit） | 基础模型降为约 25% | 小（1~2% 性能损失） | 略低但可接受 | 24GB 显卡微调 70B 模型 |

**QLoRA 关键技术：**
- **NF4（Normal Float 4）**：专为正态分布权重设计的 4bit 量化格式，比标准 INT4 精度损失更小
- **Double Quantization**：对量化常数本身再次量化，额外节省约 0.4bit/参数
- **Paged Optimizers**：优化器状态放在 CPU 内存（统一内存），避免 GPU OOM

**低显存实操适配技巧：**
```python
from transformers import BitsAndBytesConfig

bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",          # 使用 NF4 格式
    bnb_4bit_compute_dtype=torch.bfloat16,  # 计算时反量化到 BF16
    bnb_4bit_use_double_quant=True,     # 开启双重量化
)

# LoRA 配置
lora_config = LoraConfig(
    r=16, lora_alpha=32,
    target_modules=["q_proj", "v_proj"],  # 只对 Q/V 矩阵加 LoRA
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)
```

**梯度检查点（gradient_checkpointing=True）**：以重算换显存，显存再降 30%~40%

### Q3. 大模型训练/微调常见报错与整改方案

| 问题 | 根因 | 整改方案 |
|------|------|---------|
| **Loss 不下降** | ① 学习率过小/过大 ② 数据格式错误（答案被 mask 成 label=-100）③ 模型冻结层配置错误 | ① 尝试 lr=1e-4~5e-5（LoRA 微调标准范围）② 检查 DataCollator 的 labels 设置 ③ 确认 LoRA 参数 requires_grad=True |
| **梯度爆炸（loss=NaN）** | ① lr 过大 ② 数据存在异常值（如无穷大） ③ BF16 溢出 | ① 降低 lr 10x ② 数据预处理过滤异常 ③ 开启梯度裁剪 max_norm=1.0，关键 loss 用 FP32 |
| **样本拟合失效（特定类别不学）** | ① 训练数据分布极度不均衡 ② Template 格式不一致导致 tokenize 异常 | ① 对少数类过采样 / 加权损失 ② 统一 Prompt Template，打印 tokenize 结果验证 |

### Q4. 单机多卡 vs 分布式 DDP 微调

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 单机 4~8 卡，模型可单卡加载 | DDP（PyTorch DistributedDataParallel） | 每卡持有完整模型副本，通信开销仅梯度同步，实现简单 |

3. 设置 `find_unused_parameters=False`（DDP 配置），减少通信开销

### Q1. 余弦相似度计算公式与 Embedding 匹配逻辑

**余弦相似度公式：**

$$\cos(\theta) = \frac{\vec{A} \cdot \vec{B}}{\|\vec{A}\| \cdot \|\vec{B}\|} = \frac{\sum_{i=1}^{n} A_i B_i}{\sqrt{\sum_{i=1}^{n} A_i^2} \cdot \sqrt{\sum_{i=1}^{n} B_i^2}}$$

**核心代码逻辑：**

```python
import numpy as np

def cosine_similarity(vec_a: list, vec_b: list) -> float:
    """计算两个向量的余弦相似度"""
    a = np.array(vec_a)
    b = np.array(vec_b)
    
    dot_product = np.dot(a, b)           # 点积
    norm_a = np.linalg.norm(a)           # L2 范数
    norm_b = np.linalg.norm(b)           # L2 范数
    
    if norm_a == 0 or norm_b == 0:
        return 0.0                        # 零向量处理
    
    return dot_product / (norm_a * norm_b)

def batch_similarity_search(query_vec: list, doc_vecs: list, top_k: int = 5):
    """批量检索最相似的 Top-K 文档"""
    query = np.array(query_vec)
    docs = np.array(doc_vecs)            # shape: (N, dim)
    
    # 批量计算：矩阵乘法 + 归一化（高效向量化）
    query_norm = query / np.linalg.norm(query)
    docs_norm = docs / np.linalg.norm(docs, axis=1, keepdims=True)
    
    scores = docs_norm @ query_norm       # shape: (N,)
    top_k_indices = np.argsort(scores)[::-1][:top_k]
    
    return [(idx, scores[idx]) for idx in top_k_indices]
```

**取值范围：**[-1, 1]，RAG 场景中通常阈值设为 0.7~0.8 作为相关性门槛

### Q2. 批量文档清洗、去重、过滤低质量文本

```python
import re
import hashlib
from typing import List

def clean_text(text: str) -> str:
    """文本清洗"""
    text = re.sub(r'\s+', ' ', text)          # 合并多余空白
    text = re.sub(r'[^\w\u4e00-\u9fff，。？！、：；""''（）【】\s]', '', text)  # 去除乱码特殊字符
    text = text.strip()
    return text

def dedup_by_hash(texts: List[str]) -> List[str]:
    """基于哈希精确去重"""
    seen = set()
    unique = []
    for text in texts:
        h = hashlib.md5(text.encode()).hexdigest()
        if h not in seen:
            seen.add(h)
            unique.append(text)
    return unique

def filter_low_quality(texts: List[str], min_len=50, max_len=2000) -> List[str]:
    """过滤低质量文本"""
    result = []
    for text in texts:
        # 过滤过短/过长
        if not (min_len <= len(text) <= max_len):
            continue
        # 过滤中文比例过低（乱码/英文垃圾）
        chinese_ratio = len(re.findall(r'[\u4e00-\u9fff]', text)) / len(text)
        if chinese_ratio < 0.3:
            continue
        # 过滤重复字符（如"哈哈哈哈哈"类噪声）
        if max(text.count(c) for c in set(text)) / len(text) > 0.5:
            continue
        result.append(text)
    return result

# 完整 Pipeline
def process_documents(raw_texts: List[str]) -> List[str]:
    texts = [clean_text(t) for t in raw_texts]
    texts = dedup_by_hash(texts)
    texts = filter_low_quality(texts)
    return texts
```

### Q3. 训练集/测试集分布不一致的修正方案

**算法层面：**

1. **重要性采样（Importance Weighting）**：估计训练集和测试集的分布差异比值，为训练样本赋予权重，使加权训练分布向测试分布靠拢（核心方法：KLIEP、KMM）
2. **域适应（Domain Adaptation）**：使用 MMD 损失（最大均值差异）或对抗训练让模型特征在两个分布上对齐（DANN 方法），使提取的特征对分布偏移鲁棒

**数据层面：**

1. **重采样**：分析测试集数据特征（话题分布、时间分布、风格分布），在训练集中上采样与测试集分布接近的样本，下采样偏离分布的样本
2. **数据增强补充**：针对测试集中覆盖不足的子分布，使用数据增强或 LLM 生成合成数据来填补分布空白，直接减少分布 Gap

### Q4. Embedding 模型好坏评估：工业界量化评估指标 + 实测流程

**核心评估指标：**

| 指标 | 含义 | 计算方式 |
|------|------|---------|
| **Recall@K** | Top-K 检索中命中正确文档的比例 | Recall@5 = 命中数 / 总查询数 |
| **MRR（Mean Reciprocal Rank）** | 第一个正确结果排名的倒数均值 | $MRR = \frac{1}{|Q|}\sum_i \frac{1}{rank_i}$ |
| **NDCG@K** | 考虑排序位置的归一化折损累计增益 | 相关文档排名越靠前分数越高 |
| **语义相似度相关性** | 与人工标注相似度分数的 Spearman 相关系数 | STS 基准测试标准指标 |

**工业界实测流程：**
```
Step 1: 构建业务评估集
  - 从业务文档中抽取 200~500 个问题
  - 人工标注每个问题对应的正确文档段落（golden passage）

Step 2: 基准测试
  - 对评估集中每个问题，使用 Embedding 模型检索 Top-10
  - 计算 Recall@1/3/5/10、MRR@10

Step 3: 对比测试
```
