# FLAT
**核心原理**
**不做任何索引结构，原始向量原封不动存储，查询时暴力枚举计算所有距离**。
```
存储结构：
┌─────────────────────────────────────┐
│  vec_0: [0.1, 0.3, 0.5, ..., 0.2]  │
│  vec_1: [0.4, 0.1, 0.8, ..., 0.6]  │
│  vec_2: [0.2, 0.7, 0.3, ..., 0.9]  │
│  ...                                │
│  vec_n: [0.6, 0.2, 0.4, ..., 0.1]  │
└─────────────────────────────────────┘
查询过程：
query → 与 vec_0 计算距离
      → 与 vec_1 计算距离
      → 与 vec_2 计算距离
      → ...全部算完
      → 排序取 TopK
```
**适用场景**
场景1：数据量小，精度要求极高（金融风控、医疗诊断）
场景2：作为其他索引召回率评测的 Ground Truth
场景3：离线批处理，不在乎延迟
场景4：原型验证阶段，快速跑通流程
**不适合用 FLAT 的场景**
❌ 数据量超过100万 
❌ 在线服务有延迟 SLA（如 P99 < 50ms）
❌ 内存资源紧张
❌ 需要频繁更新/删除数据的场景
