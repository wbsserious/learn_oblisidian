# milvus数据库
Milvus 是**开源向量数据库**
	**存向量 → 快速查最相似的向量**
**核心特性：** **高性能向量检索** 支持十亿级向量的毫秒级 ANN（近似最近邻）搜索，底层集成了 FAISS、HNSW、DiskANN 等主流索引算法。
在 RAG 里：
1. 把文档切成片段
2. 转成**向量（embedding）**
3. 存进 Milvus
4. 用户提问 → 转向量 → 检索最相关片段
5. 丢给 LLM 生成答案
**Milvus = RAG 的 “记忆大脑”**
## 核心架构
### 父块（Parent Document / 父分片）
#### 作用
**存储完整的原始文档数据**
#### 特点
- 块很大（512~2048 字符）
- **不生成向量**
- 不参与检索
- 只在查到子块后，把**父块完整内容**给大模型
### 子块（Child Chunk / 子分片）
#### 作用
**专门用来做向量检索**
- 从父块里切出来的**小碎片**
- 生成向量
- 负责被搜索、匹配、召回
#### 特点
- 块很小（128~512 字符）
- **必须生成向量**
- 相似度检索速度极快
- 不直接给 LLM，只用来找到对应的父块
# 属性特征
- `FLAT` — 精确搜索，适合小数据集
- `IVF_FLAT / IVF_SQ8` — 倒排索引，平衡速度与精度
- `HNSW` — 图索引，高召回率
- `DiskANN` — 磁盘索引，支持超大规模数据
