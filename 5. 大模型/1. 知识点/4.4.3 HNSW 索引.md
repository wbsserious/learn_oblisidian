# 简介
## 理论基石
1. 小世界网络理论（Watts-Strogatz, 1998）
	任意两个节点之间只需经过少数几跳即可连通（"六度分隔"）。其数学特征是：
	$$L \propto \frac{\ln N}{\ln \ln N}$$
	其中$L$ 是网络直径（最短路径的期望长度），$N$ 是节点数。这意味着即使$N$达到十亿级，$L$ 也只有个位数量级。
2. NSW 图（Navigable Small World, Malkov 2014）
	将小世界网络引入向量搜索：每个向量是图中的一个节点，相似向量之间连边。搜索时从某个入口节点出发，贪心地沿边走向更近的节点，直到无法继续改进。
	NSW 的问题是**"早期贪心陷阱"**：在图的稀疏区域，搜索可能过早陷入局部最优，距离真实最近邻还很远时就停止了。**

# 构建流程
## 节点插入算法（构建期）
HNSW 的图质量完全由插入算法决定，这是理解整个索引的关键。
### 第一步：确定新节点的最大层级、
每次插入新节点时，独立地从指数分布采样一次，新节点$v$ 的最高层 $l$ 由指数概率分布采样：
$$
l=⌊−ln(uniform(0,1))⋅m_L​⌋,m_L​=\frac{1}{lnM}​
$$
其中$M$是每节点最大连接数。这个设计使高层节点数量按指数衰减：
$$\mathbb{E}[第 i 层节点数]=N⋅e^{−i⋅lnM}=N/M^i$$
- $N$向量总数量
### 第二步：从顶层贪心搜索插入位置
**（$L$ 到 $l+1$）：快速定位，不建边**
```
从最顶层入口点出发
对每一层 lc 从 L 到 l+1：
    在当前层贪心搜索，找到距 v 最近的 1 个节点（ef=1）
    将该节点作为下一层的入口点
```
新节点 $v$ 的最高层是$l$，所以$l+1$及以上的层根本不包含$v$。这些层的唯一作用是 **导航**——利用稀疏的高层图快速跳到向量空间中离$v$近的区域，找到一个好的入口点，传递给阶段二使用。ef=1 意味着只追踪最近的一个节点，速度最快，精度要求也低，因为目的只是"定位区域"而非"找所有邻居"
### **第三步：邻居选择的启发式算法（HNSW 的精髓）**
**（$min(L,l)$ 到 0）：真正建边**
```
对每一层 lc 从 min(L, l) 到 0：
    在当前层用 ef=efConstruction 搜索最近的候选集 W
    从 W 中选择 M 个邻居与 v 连边（启发式选择）
    对每个被选邻居，若其度数超过 Mmax，裁剪其连接
```
插入的核心工作：
为$v$在它实际存在的这一层建立连接。
在每一层用更大的候选集（ef=efConstruction）搜索，找到足够多的候选邻居，然后用启发式算法从中挑出最优的$M$个与 $v$ 连边。最后还要反向检查：被连接的邻居如果因此超出了最大度数限制$Mmax$，需要裁剪掉它们质量最差的一条旧边，保证整张图不会因为某些节点度数过高而退化。
HNSW 使用**启发式选择（Algorithm 4 in 原论文）**
新选择的邻居需要满足：$e$ 到查询点$q$ 的距离，必须小于$e$ 到所有已选邻居的距离。
这个条件的几何意义：确保选出的邻居在空间上尽量分散，避免所有邻居聚集在同一方向，从而保证图的全局可导航性。
## 搜索算法（查询期）
**阶段一（第 L 层到第 1 层）：粗定位，ef=1**
在稀疏的高层图中，每一层只找"当前最近的 1 个节点"然后下沉。因为高层节点稀疏，每层的邻居数量少，贪心移动的代价极低。这一阶段的目的不是找到最终答案，而是把入口点从"图的任意位置"快速收缩到"查询向量附近的区域"。
**阶段二（第 0 层）：精搜索，ef 可调**
第 0 层包含全量节点，图最密集。此时入口点已经在目标附近，用大小为 `ef` 的候选集做扩展搜索，找到真正的 TopK 近邻。
搜索算法的时间复杂度为：
$$O\left(\frac{\log N}{\log M} + ef \cdot M \cdot d\right)$$
前项是层数（对数级），后项是第 0 层的精细搜索代价。

# 关键参数：
**参数 $M$：每节点最大连接数**
$M$同时决定两件事：图的连通质量，以及内存开销。
$M$与召回率的关系：$M$ 增大 → 每个节点连接更多邻居 → 图更密 → 召回率提升，但内存线性增长。经验最优区间：$M ∈ [8, 64]$，大多数场景$M=16$ 已足够。
内存开销：
每个节点在第 0 层存 2M2M 2M 条边（入边+出边），在上层存 MM M 条边。内存估算：
$$\text{内存} = N \times (d \times 4 + 2M \times 8 + M \times 8 \times \frac{1}{\ln M}) $$
简化为经验公式：
$$
\text{内存} \approx N \times d \times 4 \times (1 + \frac{2M}{d})
$$
**参数 efConstruction：构建质量**
构建时的候选集大小，影响图结构的质量（离线一次性代价）：
$$\text{构建时间} \propto N \times efConstruction \times M \times d$$
约束条件：$efConstruction \geq 2M$，否则搜索候选不足，图质量退化。推荐值：$efConstruction = 200 \sim 400$,超过 500 后边际收益极小。
**参数 ef：查询时候选集**
这是查询期唯一可调参数，直接控制召回率和延迟的权衡：$$\text{查询延迟} \propto ef \times M \times d$$
约束条件：ef≥Kef \geq K ef≥K（返回数量），否则候选不足，实际返回少于 K 个。
```
ef=K       → Recall ~85%，最快
ef=2K      → Recall ~92%
ef=4K      → Recall ~97%
ef=10K     → Recall ~99%
ef=∞       → Recall=100%（退化为暴力搜索）
```
# **固有局限性**
**内存不可压缩：** HNSW 必须在内存中同时存储原始向量（用于距离计算）和图结构（用于导航），无法像 IVF_PQ 那样压缩向量。这是选择 DiskANN 的根本原因。
**构建不可增量删除：** 删除节点后图结构不会自动修复，孤立的"幽灵节点"会降低搜索质量。Milvus 通过软删除 + 定期 Compaction 重建图来解决这个问题。
**高维稀疏向量失效：** 维度 d>1000，d > 1000，d>1000 且向量稀疏时，HNSW 的邻居选择启发式失效——因为高维稀疏空间中距离函数本身的区分度下降。此时应考虑稀疏索引（SPARSE_INVERTED_INDEX）。
**图结构无法共享：** 与 IVF 的聚类中心不同，HNSW 的图结构是全局耦合的，无法分片独立构建后合并，限制了超大规模的分布式构建能力。
# HNSW vs IVF 的本质差异
```
              HNSW                      IVF
核心数据结构   多层有向图               倒排列表 + 聚类
搜索策略       图上贪心游走             聚类剪枝 + 暴力搜索
误差来源       局部最优（图密度不够）   边界误差（跨簇最近邻）
召回率控制     ef 参数                  nprobe 参数
内存模式       向量 + 图（难压缩）      向量可量化（IVF_PQ）
更新友好性     差（删除破坏图结构）     较好（重新聚类即可）
高维适应性     中等（M 值影响显著）     差（维度灾难更明显）
最优规模       百万级                   百万~十亿级
```
**精度优先且内存充足 → HNSW**
**规模优先或内存受限 → IVF_PQ**
**超大规模磁盘场景 → DiskANN**
	（DiskANN 本质上是将 HNSW 的图结构卸载到 SSD 的工程实现）。
