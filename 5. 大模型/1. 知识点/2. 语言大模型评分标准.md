# N-gram 匹配类

| 指标         | 核心思想                    | 典型场景   |
| ---------- | :---------------------- | :----- |
| **BLEU**   | n-gram 精确率              | 机器翻译   |
| **ROUGE**  | n-gram 召回率（ROUGE-1/2/L） | 文本摘要   |
| **METEOR** | 精确率+召回率+同义词匹配           | 翻译、摘要  |
| **CIDEr**  | 对高频 n-gram 降权（TF-IDF思想） | 图像描述生成 |
## BLUE指标
公式：
$$
BLUE = BP·exp{\left(\Sigma_{n=1}^Nw_nlogP_n\right)}$$
- 默认权重：$w_n=1/N$(通常N=4)
- $p_n$:`n-gram`的剪枝精确率
- $$p_n=\frac{candidate 中与 reference 匹配的 n-gram 数量​}{candidate 中 n-gram 总数}$$
- $w_n$:`n-gram`的权重（通常是均等分配）
- $BP$：惩罚因子定义为:$$BP=\begin{cases}1\ \ &if \ c>r​\\e^{(1-r/c)}&if \ c\leq r\end{cases}$$
- $c$：候选翻译（机器生成)文本的长度
- $r$：参考翻译（真实翻译）中与候选翻译长度最接近的那个参考翻译的长度
```python
from nltk.translate.bleu_score import sentence_bleu, SmoothingFunction
def get_blue_score(fegerence,candiate,weight=(0.25, 0.25, 0.25, 0.25)):
"""  
计算BLEU相似度分数（支持自定义n-gram权重，默认BLEU-4）  
:param reference: 参考文本列表（真实结果，[[分词列表]]）  
:param candidate: 候选文本列表（模型结果，[分词列表]）  
:param weights: n-gram权重，默认(0.25,0.25,0.25,0.25)对应BLEU-4  
:return: bleu分数  
"""  
# 加入平滑函数，解决短文本高阶n-gram为0的警告
smoothie = SmoothingFunction().method4
return sentence_bleu(fegerence,candiate,weight=weigth,smoothing_function=smoothie)
```
## ROUGE评估
ROUGE（Recall-Oriented Understudy for Gisting Evaluation）是专为**文本摘要**设计的评估指标，与 BLEU 互补——BLEU 关注精确率，ROUGE 关注**召回率**，即参考文本中有多少内容被模型覆盖到了。
**ROUGE-1**：单词级别的召回率，统计候选文本中有多少单词出现在参考文本中。粒度最粗，对词序不敏感。
**ROUGE-2**：二元组（bigram）匹配，要求相邻两词一起出现在参考文本中，比 ROUGE-1 更关注局部词序和流畅度。
**ROUGE-L**：基于最长公共子序列（LCS），不要求词连续，只要顺序一致就算匹配，能捕捉句子级别的结构相似度。
计算参数：
$$
P=\frac{匹配数}{候选文本长度},R=\frac{匹配数}{参考文本长度},F1=\frac{2·P·R}{P+R}
$$
python参考代码
```python
from rouge import Rouge
rouge=Rouge()
candidate = "The brown fox jumps over the lazy dog"  
reference = "The quick brown fox jumps over the lazy dog"  
result = rouge.get_scores(candidate, reference)  
print("整体评分结果",result)  
print("rouge-1结果",result[0]['rouge-1'])  
print("rouge-2结果",result[0]['rouge-2'])  
print("rouge-l结果",result[0]['rouge-l'])
```
## PPL评估（preplexity)
PPL 是衡量语言模型**对文本预测能力**的核心指标
公式：
$$\begin{align}log P(w₁...wₙ) &= Σ log P(wᵢ | w₁...wᵢ₋₁)\\H &= − (1/N) × Σ log P(wᵢ | context)\\PPL &= exp(H) = exp(− (1/N) Σ log P(wᵢ))\end{align}$$
- $P(wₙ | w₁, w₂, ..., wₙ₋₁)$：语言模型对每个 token 给出条件概率
- $log P(w₁...wₙ)$：整句的对数概率
- $H$：每个 token 的平均负对数概率（交叉熵 H）
- $PPL$：e 的 H 次方（或 2 的 H 次方，取决于 log 底数）
	$PPL$ 可理解为模型在每一步平均需要从多少个等概率候选词中猜测下一个词。
### Perplexity per Token / Per Word
- **全称**：Perplexity per Token / Per Word
- **含义**：模型对**每个 token / 词**的平均困惑度
- **公式**：$$PPL=exp(−\frac{1}N​∑_{i=1}^N​logP(w_i​∣w<i​))$$其中：
    - $N$：总 token 数
    - $P(w_i​∣w<i​)$：模型对第 $i$ 个 token 的条件概率
- **特点**：**最通用、最标准**的困惑度指标，消除文本长度影响，便于跨模型 / 数据集比较
### PPS (Perplexity per Sentence)
- **全称**：Perplexity per Sentence
- **含义**：模型对**每个句子**的平均困惑度
- **公式**：$$PPL=exp(−\frac{1}M​∑_{i=1}^M​logP(S_j​))$$其中：
    - $M$：句子总数
    - $P(S_j​)$：模型对第 j 个完整句子的联合概率
- **特点**：**不常用**，受句子长度影响大，长句子天然 PPS 更高，可比性差
### PPW (Perplexity per Word)
- **全称**：Perplexity per Word
- **含义**：按 **词（word）** 归一化的困惑度
- **公式**：与 PPL 完全一致，仅将 N 定义为**词数**而非 token 数
- **特点**：在**词级语言模型**（如传统 n-gram）中与 PPL 等价；在**子词 token 化**（如 BPE）场景下，PPW 强调按 “词” 统计，与按 “token” 统计的 PPL 数值可能不同
