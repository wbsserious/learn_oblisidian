# **数学基础**：
IVF（Inverted File Index，倒排文件索引）的核心思想来自**Voronoi 图**的空间划分理论。
# **Voronoi 划分**：
给定 $k$ 个聚类中心 $\{c_1, c_2, ..., c_k\}$，整个向量空间被划分为 $k$ 个 Voronoi 单元：
$$V_i​={x∈R^d:∥x−c_i​∥≤∥x−c_j​∥,∀j\neq i}$$
每个向量 $x$ 归属于距离最近的聚类中心所在的单元。查询时只搜索查询向量所在单元附近，跳过远处单元，实现**空间剪枝**。

## 查询的近似误差来源
IVF 是 ANN（近似最近邻）而非精确搜索，误差来自一个假设：
> **假设：查询向量的真实最近邻，大概率与查询向量在同一个 Voronoi 单元内。**

这个假设在高维空间中并非总成立，原因是**维度灾难**：
在高维空间中，查询向量的真实最近邻有相当概率落在相邻 Voronoi 单元而非自身所在单元，这一概率随维度升高而增大，这是 nprobe 必须大于 1 的根本原因。

# IVF_PQ：乘积量化的数学原理
IVF_PQ 在 IVF 的基础上，对每个簇内的向量做**乘积量化（Product Quantization）**压缩存储。**
**核心思想：** 将高维向量分解为若干低维子空间，每个子空间独立量化。
**存储思路：**
设原始的存储向量$x \in \mathbb{R}^d$, 将其分割为$m$个子向量:
$$
x=\left[x^{(1)},x^{(2)}... x^{(m)}\right],x^{(i)}  \in \mathbb{R}^{\frac{d}m}
$$
对每个子空间，设立一个码本（codebook）$\mathcal{C}^{(j)}=\left\{\mathcal{c}^{(j)}_{1},...,\mathcal{c}^{(j)}_{2^{nbits}} \right\}$量化编码为：
$$
	\hat{x}= \left[q^{(1)},q^{(2)},...,q^{(m)}\right],q^{(j)} \in \{0,...,2^{nbits}-1\}
$$
压缩率计算：
$$\begin{align}
原始大小=d×4 字节 \ \ 压缩后=m×\frac{nbits}8​ 字节 \\
\text{压缩比} = \frac{d \times 4}{m \times nbits/8} = \frac{32d}{m \times nbits}​
\end{align}$$
- $d$：原始数据的维度
- $nbits$：number of bits，量化位数
- $m$：子空间个数
**非对称距离计算（ADC）：** 
查询时不压缩查询向量，而是预计算查询向量每个子空间与所有码字的距离，构成距离表，再通过查表累加近似距离——避免了两端都有量化误差的对称计算，精度更高。
$$
\hat{d}(q,x)=∑_{j=1}^m​d(q^{(j)},c_{q^{(j)}(x)}^{(j)}​)
$$
# IVF_SQ8：标量量化
相比 PQ 的复杂分组，SQ8 更为直接：将每个维度的 float32（4字节）映射到 uint8（1字节）。
对第$i$维，线性量化为：
$$\hat{x}_i​=round(\frac{x_i​−min_i​​}{max_i​−min_i}​×255)$$
距离计算时反量化回 float：
$$x^i​≈\hat{x}^i​×\frac{max_i​−min_i}{255​​}+min_i​$$

**特点：** 压缩比固定 4:1，实现简单，精度损失小（通常 Recall 降低 < 2%），是 IVF_FLAT 与 IVF_PQ 之间的折中选择。
# nlist 的选择
`nlist` 不是拍脑袋的参数，有理论支撑：
**经验公式（来自 FAISS 文档）：**
$$nlist≈4\sqrt{N}​$$
构建质量（聚类的 Voronoi 精度）随 `nlist` 增大而提高，直到$nlist \approx \sqrt{N}$ 时边际收益递减。乘以系数 4 是实践中精度与构建时间的经验平衡点。

| 数据量    | 推荐 nlist | 备注       |
| ------ | -------- | -------- |
| 10 万   | 512      | 小数据可适当减少 |
| 100 万  | 2048     | 经典配置     |
| 1000 万 | 8192     |          |
| 1 亿    | 65536    | 需要更多训练数据 |
# 缺点
**1. 训练集依赖性**:
K-Means 在训练集上拟合 Voronoi 划分，若线上数据分布与训练时偏移（Distribution Shift），聚类质量下降，召回率恶化。Milvus 建议训练数据量$\geq 256 \times nlist$。
**2. 高维稀疏问题**:
维度$d > 512$时，所有向量到聚类中心的距离趋于相等（维度灾难），Voronoi 划分失去区分度。此时 HNSW 的图结构比 IVF 的欧氏划分更鲁棒——这是选择 HNSW 的核心理由之一。
