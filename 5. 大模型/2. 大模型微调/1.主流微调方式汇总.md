# LoRA（最主流）
在原始权重矩阵旁边插入两个低秩矩阵 A 和 B，训练时只更新 A、B，推理时将 W+ABW + AB W+AB 合并回原矩阵，**不增加推理延迟**。
```
原始权重: W (冻结)
新增参数: W' = W + α·(B×A)
可训练参数量: 约为原来的 0.1%~1%
```
衍生变体：
- **QLoRA**：在 LoRA 基础上将基座模型量化为 4bit，显著降低显存，消费级 GPU 可训练 70B 模型
- **LoRA+**：对 A、B 矩阵设置不同学习率，提升训练效率
- **DoRA**：将权重分解为模长和方向分别更新，效果优于 LoRA
# Adapter
在 Transformer 每层的 FFN 后插入一个小型瓶颈网络（下投影→激活→上投影），只训练 Adapter 参数。**推理时有额外延迟**，这是相比 LoRA 的劣势。
# Prefix Tuning / P-Tuning v2
在每一层的 Key、Value 前面拼接可学习的"前缀向量"，不修改模型权重本身。适合生成任务，对模型结构侵入性低。
# Prompt Tuning / P-Tuning
只在**输入层**加入可学习的软提示向量，参数量极少。效果在模型规模较大时才接近全量微调，小模型上表现较弱。
# IA³
通过对激活值进行逐元素缩放来调整模型行为，可训练参数比 LoRA 更少，适合资源极度受限的场景。
# 按训练目标分类

| 方式              | 适用场景        | 代表方法        |
| --------------- | ----------- | ----------- |
| **SFT（监督微调）**   | 指令跟随、领域适配   | LoRA + 标注数据 |
| **RLHF**        | 对齐人类偏好      | PPO 算法      |
| **DPO（直接偏好优化）** | 替代 RLHF，更稳定 | 偏好对数据训练     |
| **继续预训练**       | 注入领域知识      | 全量或 LoRA    |

# 按数据类型分类
- **指令微调（Instruction Tuning）**：用"指令-回答"对训练，提升指令遵循能力
- **对话微调**：多轮对话数据，提升对话连贯性
- **领域微调**：用垂直领域语料注入专业知识

# 方法横向对比

|方法|训练参数量|显存需求|推理延迟|效果上限|
|---|---|---|---|---|
|全量微调|100%|★★★★★|无增加|最高|
|LoRA|~0.1-1%|★★|无增加|高|
|QLoRA|~0.1-1%|★|略增|接近LoRA|
|Adapter|~1-5%|★★|略增|中高|
|Prefix Tuning|<1%|★|略增|中|
|Prompt Tuning|极少|★|无增加|中低|
