# LRU 算法详解
**LRU（Least Recently Used，最近最少使用）** 是一种经典的缓存淘汰算法，核心思想是：**如果一个数据最近被访问过，那么将来被访问的概率也更高；反之，最久没被访问的数据优先淘汰。**
# 核心原理
**LRU 的核心假设：最近被用过的数据，近期再次被用的可能性更高。**
所以缓存中始终保留"最近活跃"的数据，一旦缓存满了，就把最久没被访问的那个踢掉。
# Redis 中的 LRU
Redis 并非实现了严格的 LRU，而是使用了**近似 LRU（Approximate LRU）**：
- 每次淘汰时，随机采样 `N` 个 key（默认 5 个）
- 从中选出最久未访问的那个淘汰
- 避免了维护全局链表的性能开销
Redis 4.0 之后还引入了 **LFU（Least Frequently Used）**，它不仅看最近一次访问时间，还统计访问频率，对于"偶尔访问一次的冷数据"防护性更强。
---
## LRU vs LFU 对比

|维度|LRU|LFU|
|---|---|---|
|淘汰依据|最久未访问|访问频率最低|
|实现复杂度|中等|较高|
|缓存污染|一次偶发访问即可保留|频率低的数据优先淘汰|
|适合场景|时间局部性强的访问|热点数据明显的场景|
