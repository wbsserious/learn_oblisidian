# 倒排列表
在 IVF 中，K-Means 把所有向量聚成 `nlist` 个簇。倒排列表就是一张**以簇编号为键、以该簇内所有向量为值**的映射表：
```
簇 0  →  [vec_3, vec_17, vec_42, vec_89, ...]
簇 1  →  [vec_1, vec_5,  vec_23, vec_66, ...]
簇 2  →  [vec_7, vec_14, vec_55, vec_71, ...]
...
簇 k  →  [vec_2, vec_9,  vec_38, vec_44, ...]
```
每条向量只归属于**一个**簇（距离最近的那个），所以每条向量在整张表中只出现一次。

---
# 为什么叫"倒排"
这个名字借用自**文本检索**领域的倒排索引（Inverted Index）：
```
文本倒排索引：
  "苹果" → [文档3, 文档7, 文档15]   ← 词 → 包含该词的文档列表

向量倒排列表：
  簇 2   → [vec_7, vec_14, vec_55]   ← 簇 → 归属该簇的向量列表
```
两者结构完全一致，都是"把某个 key 对应的所有条目聚合在一起"，方便查询时只扫描相关的那一组，跳过无关的其他组。
