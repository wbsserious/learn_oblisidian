Non-Autoregressive Generation (NAR)
# 非自回归生成（Non-Autoregressive Generation, NAR）
---
## 一、自回归生成的背景与局限
在介绍 NAR 之前，需先理解其对立面——**自回归生成（Autoregressive Generation, AR）**。
标准的序列生成模型（如 GPT、Transformer decoder）将生成过程建模为条件概率的链式分解：
$$P(y_1, y_2, \ldots, y_T \mid x) = \prod_{t=1}^{T} P(y_t \mid y_{<t}, x)$$
其中 $y_{<t} = (y_1, \ldots, y_{t-1})$ 为已生成的历史 token，$x$ 为输入条件（如源语言句子）。
**AR 的核心瓶颈：串行解码（Sequential Decoding）**
- 每个 token 必须等待前一个生成完毕，**时间复杂度为 $O(T)$**（$T$ 为序列长度）
- 无法利用 GPU/TPU 的并行计算能力
- 在长序列、实时推理场景下延迟（latency）极高
## 二、NAR 的核心思想
**非自回归生成**打破了 token 间的顺序依赖，将联合分布建模为**条件独立的乘积形式**：
$$P_{\text{NAR}}(y_1, \ldots, y_T \mid x) = \prod_{t=1}^{T} P(y_t \mid x, z)$$
其中 $z$ 是从输入 $x$ 提取的隐变量（latent variable），用于提供全局上下文。**所有 token 在同一时间步并行生成**，推理时间复杂度降至 $O(1)$（理论上）。
> **奠基工作**：Gu et al. (2018) _"Non-Autoregressive Neural Machine Translation"_（ICLR 2018）首次将 NAR 框架引入神经机器翻译（NMT），在速度上取得 **15×** 加速，但 BLEU 分数有明显下降。

## NAR 的核心思想
一句话：**打破"过去决定未来"的生成枷锁，让所有位置同时开口说话。**
## 从一个直觉出发
人类写作时，并不是严格从左到右、每个字都依赖前一个字才能落笔。在下笔之前，脑子里已经有了**整句话的大意**，然后各个部分"同时成形"。
AR 模型恰恰相反——它被迫模拟一个极端串行的过程：
```
"我"→ "我爱" → "我爱北" → "我爱北京"
  必须等前一步完成，才能走下一步
```
NAR 的思想就是：**能不能先形成一个全局的意图/语义，然后让每个位置并行地、同时地把词填上去？**
```
        [全局语义 z]
    ↙    ↙    ↘    ↘
   I   love   Bei  jing     ← 同时生成
```
---
## 思想的本质：一次"独立性假设"的豪赌
AR 的概率分解：
$$P(y_1, y_2, \ldots, y_T \mid x) = \prod_{t=1}^T P(y_t \mid \underbrace{y_{<t}}_{\text{历史依赖}}, x)$$
NAR 的概率分解：
$$P(y_1, y_2, \ldots, y_T \mid x) = \prod_{t=1}^T P(y_t \mid \underbrace{z}_{\text{全局语义}}, x)$$
NAR 说：**我不需要知道左边的邻居是谁，只要我们都看着同一份"全局蓝图" $z$，各自独立填词就够了。**
这个假设**大胆但有代价**——它抹去了 token 之间真实存在的局部依赖关系。
## 思想的三个层次
### 第一层：并行性（Parallelism）
最直接的动机——推理速度。AR 是 $O(T)$ 串行，NAR 是 $O(1)$ 并行，在 GPU 上可以充分利用硬件。
### 第二层：全局规划（Global Planning）
AR 天生是"短视"的——它只能看到过去，无法预见未来。NAR 的隐变量 $z$ 代表了对**整个输出**的一种全局编码，某种意义上更接近人类"先有整体意图，再落笔"的认知方式。
```
AR 视角：  我已经写了"I love"，下一个词是什么？
NAR 视角：  这句话的意思是"表达对北京的喜爱"，所有位置应该填什么？
```
### 第三层：条件独立是一种结构归纳偏置（Inductive Bias）
NAR 本质上在说：**给定足够丰富的全局信息，token 之间的局部依赖是可以被"压缩"进全局表示的**。这一思想在某些任务（结构化生成、格式固定的输出）中确实成立，但在开放域创作中则严重失效。
## 思想的代价与补救
NAR 的独立性假设制造了一个根本矛盾：
```
真实世界：  "北京" 和 "capital" 都是合理翻译
NAR 模型：  左半句学到了"北京"，右半句学到了"capital"
输出结果：  "北京 capital"  ← 两个模态的混合，谁也不像
```
这就是**多模态问题**。补救思路沿着一条清晰的路径演进：
```
单步NAR（1次）
    ↓ 质量太差
迭代细化（多次修正）
    ↓ 步数仍有限制
扩散模型（从噪声逐步去噪）
    ↓ 与AR结合
Speculative Decoding（NAR起草，AR验证）
```
每一步都是在**"独立性假设"这个口子上打补丁**——引入越来越多的 token 间交互，以换取质量的提升，同时尽量保留并行带来的速度优势。
## 最深层的哲学
NAR 与 AR 的对立，映射的是序列生成中两种根本不同的世界观：

|              | AR          | NAR         |
| ------------ | ----------- | ----------- |
| **信息流向**     | 过去 → 未来（因果） | 全局 → 局部（广播） |
| **生成隐喻**     | 讲故事（一句一句）   | 填空（有了大纲再填词） |
| **对不确定性的处理** | 逐步消解        | 一次性坍缩       |
| **适合的任务**    | 开放创作、对话     | 结构化生成、翻译、编辑 |

**NAR 的核心思想，归根结底是一句话：与其让序列自己生长，不如先给它一个灵魂（全局语义），再让各个部分同时具象化。** 快，是结果；全局优先，才是思想。
## 三、核心挑战：多模态问题（Multimodality Problem）
NAR 面临的最根本挑战是**条件独立假设与真实数据分布之间的鸿沟**。
### 3.1 Multi-modality Problem
在 AR 中，$y_t$ 的生成受 $y_{<t}$ 约束，保证了序列的**内部一致性**。在 NAR 中，各 token 独立采样，容易出现：
- **重复（Repetition）**：`the the cat`
- **缺失（Omission）**：跳过关键词
- **不一致（Incoherence）**：同一概念被翻译成多个不同词
这本质上是因为目标序列存在多种合法变体（即"多模态分布"），模型在缺乏自左向右约束的情况下，倾向于对多个模态求平均，导致"模态折叠"现象。
### 3.2 信息瓶颈问题
$z$ 作为唯一的全局上下文，需要编码所有 token 生成所需的信息，这对潜变量的表达能力提出极高要求。
## 四、主要技术路线
### 4.1 迭代细化（Iterative Refinement）
不再强求一次生成完美序列，而是多轮迭代逐步修正。
**代表工作：Mask-Predict（Ghazvininejad et al., 2019）**
- 第一步：并行生成所有 token
- 随后迭代：mask 掉低置信度 token，重新预测
- 迭代 $K$ 次后输出
$$y^{(k+1)}_t = \begin{cases} \text{resample} & \text{if } t \in \text{MASK}^{(k)} \ y^{(k)}_t & \text{otherwise} \end{cases}$$
这是 **BERT-style masked language model** 在生成侧的直接应用。
### 4.2 连接时序分类（CTC, Connectionist Temporal Classification）
将生成任务转化为对**超长序列**的分类，再通过 CTC 解码折叠为目标序列。
$$P(y \mid x) = \sum_{\pi \in \mathcal{B}^{-1}(y)} P(\pi \mid x)$$
其中 $\mathcal{B}$ 为折叠函数（去重、去空白符），$\pi$ 为对齐路径。
**代表工作**：Libovický & Helcl (2018)，以及后续 GLAT + CTC 系列。
### 4.3 流模型与变分推断（Flow / VAE）
通过引入**显式潜变量 $z$** 来建模目标分布的多模态性。
**代表工作：FlowSeq（Ma et al., 2019）**
$$P(y \mid x) = \int P(y \mid z, x) P(z \mid x) , dz$$
通过可逆变换（Normalizing Flow）将简单先验分布映射为复杂的目标分布，从而在保持并行解码的同时提升生成多样性。
### 4.4 扩散模型（Diffusion Models）
将离散 token 生成建模为**连续空间中的逐步去噪过程**。
**代表工作：Diffusion-LM（Li et al., 2022），MDLM，DiffuSeq**
前向过程（加噪）： $$q(x_t \mid x_{t-1}) = \mathcal{N}(x_t; \sqrt{1-\beta_t} x_{t-1}, \beta_t \mathbf{I})$$
反向过程（去噪，并行）： $$p_\theta(x_{t-1} \mid x_t) = \mathcal{N}(x_{t-1}; \mu_\theta(x_t, t), \Sigma_\theta(x_t, t))$$
扩散模型天然支持**并行解码**，且通过多步去噪缓解了单步 NAR 的多模态问题，代价是推理步数增加（通常 10–100 步）。
### 4.5 一致性训练（Glancing Training / GLAT）
**代表工作：GLAT（Qian et al., 2021, ACL）**
核心思想：在训练时**动态地让模型"参考"部分目标 token**，以逐渐学会 token 间的依赖关系：
1. 用当前模型对目标句子做预测
2. 统计预测正确的 token 数 $n_{\text{correct}}$
3. 随机 mask 掉 $\lambda \cdot (T - n_{\text{correct}})$ 个目标 token 暴露给模型
4. 模型在"看到部分答案"的条件下预测其余 token
这一课程学习（curriculum learning）策略显著改善了模型的序列一致性，使 NAR 的 BLEU 分数大幅提升，逼近 AR 水平。
## 五、长度预测问题
NAR 需要事先确定输出序列长度 $T$，这本身就是一个非平凡的子问题。
常见解决方案：

|方法|描述|
|---|---|
|**长度预测器（Length Predictor）**|用独立分类头预测 $T$，训练时以目标长度为监督|
|**Fertility（生育率）**|每个源 token $x_i$ 预测生成 $f_i$ 个目标 token，$T = \sum f_i$（Gu et al. 2018 原始方案）|
|**CTC 超长序列**|固定输出长度为 $cT_x$（如 $c=2$），通过折叠得到可变长输出|
|**插值/推测**|用 AR 模型草稿 + NAR 模型细化（Speculative Decoding 系列）|
## 六、性能对比总结

|维度|AR|NAR（单步）|NAR（迭代）|
|---|---|---|---|
|**推理延迟**|$O(T)$|$O(1)$|$O(K)$，$K \ll T$|
|**生成质量**|★★★★★|★★★|★★★★|
|**并行度**|低|极高|高|
|**训练复杂度**|低|中高|中高|
|**适用场景**|通用生成|实时/边缘推理|高质量快速推理|
## 七、前沿方向与与 LLM 的关系
### 7.1 Speculative Decoding（推测解码）
严格意义上是 AR+NAR 的混合范式：用小模型（draft model，NAR 风格并行生成多 token）+ 大模型（AR 风格验证）： $$\text{accept } y_t \text{ if } P_{\text{large}}(y_t) / P_{\text{draft}}(y_t) \geq u, \quad u \sim \text{Uniform}[0,1]$$ 在保证与 AR 大模型**等价输出分布**的前提下，实现 2–4× 加速。
### 7.2 MDLM / Masked Diffusion LM
将离散掩码过程与扩散模型统一，是目前最具竞争力的 NAR 语言模型框架之一。
### 7.3 与 RLHF/对齐的结合
NAR 模型的快速推理为大规模在线强化学习（online RLHF）提供了可行性，是未来对齐高效化的重要研究方向。

## 八、小结
NAR 的核心贡献在于**用条件独立假设换取推理并行性**，代价是引入了多模态问题和序列一致性挑战。从 Gu et al. (2018) 的原始单步 NAR，到 Mask-Predict、GLAT、扩散模型，再到 Speculative Decoding，整个领域的演进轨迹是：**不断在"并行度"与"生成质量"之间寻找新的帕累托最优点**。
随着 LLM 部署成本和实时性需求日益突出，NAR 及其衍生技术将在工业界占据越来越重要的地位。

## 九、 NAR 在大模型中的实际应用形态
### Speculative Decoding（推测解码）⭐ 最主流

这是目前**工业界落地最广**的 NAR 应用，被 Google、Meta、Anthropic 等广泛采用。

```
小草稿模型（Draft Model）         大验证模型（Target Model）
     ↓ NAR 风格，快速并行生成        ↓ AR，但并行验证多个token
  [token1, token2, token3, token4]  → 一次前向传播批量接受/拒绝
```

**关键数学保证**：通过拒绝采样，最终输出分布与大模型 AR 解码**完全等价**：
$$
\text{accept } y_t \Leftrightarrow u < \frac{P_{\text{large}}(y_t \mid \cdot)}{P_{\text{draft}}(y_t \mid \cdot)}, \quad u \sim U[0,1]
$$
**实际落地案例**:

|产品/框架|实现方式|加速比|
|---|---|---|
|Gemini（Google）|小模型草稿 + 大模型验证|2~3×|
|LLaMA 3（Meta）|Self-speculative / Medusa|2~4×|
|TensorRT-LLM|内置 speculative decoding|2~3×|
|vLLM|支持多种 draft 策略|可配置|
### Medusa：大模型长出"多头"

**核心思想**：在原始 LLM 上附加多个并行预测头，每个头预测未来第 kk k 步的 token：

```
                    LLM 主干（frozen）
                         ↓
             hidden state h_t
           ↙      ↓       ↓      ↘
        head_1  head_2  head_3  head_4
          ↓       ↓       ↓       ↓
        y_{t+1} y_{t+2} y_{t+3} y_{t+4}   ← 并行预测未来4步
```

- 额外头只需要微调，主干不动
- 用 Tree Attention 对多个候选路径同时验证
- 实测在 Vicuna-7B 上加速 **2.2~3.6×**
### EAGLE / EAGLE-2：更激进的草稿生成
2024 年提出，草稿模型不只看 token，还复用大模型的**特征向量（feature）**：
```
大模型:  x → [f_1, f_2, f_3] → y
                    ↓
草稿模型: 复用 f_t + token embedding → 预测 y_{t+1}, y_{t+2}...
```
由于草稿模型获得了更丰富的上下文信息，接受率大幅提升，实测加速 **3~4×**，是目前 speculative decoding 的 SOTA 之一。
### Diffusion LLM（扩散式大语言模型）
这是最接近"真正 NAR 大模型"的方向，2024~2025 年出现了几个重要工作：
**MDLM / SEDD（离散扩散）**
```
前向过程（训练）：句子 → 逐步 mask → 全 [MASK]
反向过程（推理）：全 [MASK] → 逐步去噪 → 完整句子
                  ↑ 每步都是并行的！
```
**LLaDA（Large Language Diffusion with mAsking，2025）**
北大提出，直接在大规模数据上训练 8B 参数的 Masked Diffusion LM：
- 推理时从全 mask 开始，迭代去噪 ~10 步
- 在部分任务上逼近同参数量的 AR 模型
- 在**计划性强的任务**（如数独、代码填空）上甚至**超越 AR**

```
AR 的弱点：           NAR/扩散的优势：
只能从左到右          可以先确定关键位置
无法"反悔"           可以在全局视角下迭代修正
```

**Mercury（Inception Labs，2025）**

首个商业化 Diffusion LLM，定位为"快速、廉价"的代码生成模型，主打推理速度。

### 2.5 Jacobi Decoding / Lookahead Decoding

一种无需额外模型的 NAR 加速方法：

```
普通 AR：     y1 → y2 → y3 → y4  （串行）

Jacobi：      随机初始化 [y1', y2', y3', y4']
              同时用大模型并行验证/更新所有位置
              迭代直到收敛（通常 < T 步）
```

不需要任何额外训练，是一种**即插即用**的推理加速方案，但加速比相对有限（1.5~2×）。
