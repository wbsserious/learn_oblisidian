DeepSeek（深度求索）模型以**仅解码器 Transformer**为基础，核心演进为**MoE 混合专家架构**，并叠加**MLA 多头潜在注意力、RoPE、GQA、MTP**等自研优化，形成兼顾**超大容量、高效推理、长文本能力**的技术体系。
## 一、基础架构：仅解码器 Transformer（通用底座）
DeepSeek 沿用**Decoder-only Transformer**作为底层骨架，与 GPT 同源但做了大量工程与算法优化。
### 1. 输入层
- **Token 嵌入（Embedding）**：将文本转为高维向量（如 4096/8192 维）。
- **位置编码**：采用**旋转位置编码（RoPE）**，支持超长上下文（最高 128K token）。
### 2. 标准 Transformer 解码器层（密集版）
每一层包含：
1. **多头潜在注意力（MLA）**：DeepSeek 自研，替代标准多头注意力（MHA）。   
    - 核心：通过**低秩压缩**将 Q/K/V 投影到更小潜在空间，再恢复，大幅降低 KV Cache 显存占用（约为标准 MHA 的 1/3）。
    - 搭配**分组查询注意力（GQA）**：多查询头共享 K/V 头，进一步提升长文本推理效率。    
2. **RMSNorm**：替代 LayerNorm，训练更稳定、收敛更快。
3. **前馈网络（FFN）**：在 MoE 版中被**MoE 层**替换。
4. **残差连接**：贯穿注意力与 FFN/MoE 模块。
### 3. 输出层
- 线性层 + Softmax：映射到词表概率，自回归生成下一个 token。
## 二、核心架构：DeepSeek-MoE（混合专家，V2/V3 主力）
### 1. MoE 核心思想（稀疏激活）
- **总参数极大、激活参数极小**：
        - 例：DeepSeek-V2 总参 236B，**每 token 仅激活约 21B**（≈1/10）。
    - 实现 “**大模型容量 + 小计算开销**” 的平衡。
### 2. MoE 层结构（标准配置）
- **专家数量**：64/128/256 个 FFN 专家 + 1 个共享专家。
- **路由机制**：
      - 门控网络（Gate）为每个 token 计算专家权重，**Top-K 激活**（通常 K=8）。
    - 采用**负载均衡损失**，避免 “专家塌缩”（少数专家被过度激活）。
- **计算流程**：
    输入 → 门控选专家 → 激活 K 个专家并行计算 → 加权融合 → 输出。
### 3. MoE 关键优化（DeepSeek 自研）
- **专家分层 / 领域化**：按语义、推理、代码、数学等划分专家簇。
- **动态路由缓存**：降低跨设备通信开销。
- **共享专家**：处理通用模式，减少冗余计算。
## 三、三大自研核心技术（效率与性能突破）
### 1. 多头潜在注意力（MLA）
- 解决：长文本 KV Cache 显存爆炸。
- 原理：低秩压缩 + 潜在空间注意力，显存占用降至标准 MHA 的 1/3，推理速度提升 1.8 倍。
### 2. 多 Token 预测（MTP）
- 训练目标：不只预测下一个 token，同时预测后续 N 个 token。
- 收益：**训练信号密度提升**，收敛更快、生成质量更稳。
### 3. 全栈工程优化
- **精度**：FP8 混合精度训练，显存与算力开销大幅降低。
- **并行**：5D/8D 混合并行（数据 / 模型 / 流水线 / 专家 / 序列 / 通信等），支持万亿参数训练。
- **推理**：Prefill/Decode 分离、FlashAttention-2、KV Cache 量化等。

## 五、训练范式（预训练 + 对齐）
1. **预训练**：
    - 目标：自回归语言建模 + MTP 多 token 预测。
    - 数据：海量高质量文本（含代码、学术、多语言）。
    - 策略：分阶段预训练、动态数据混合。
2. **对齐**：
    - 监督微调（SFT）：指令数据。
    - RLHF：人类反馈强化学习，提升安全性与有用性。
