# RLHF 详解：基于人类反馈的强化学习
## 一、背景与动机
### 为什么需要 RLHF？
预训练语言模型（如 GPT）的目标是**最大化下一个 token 的预测概率**，本质上是在做"文本补全"。这与人类真正想要的——**有帮助、无害、诚实的回答**——存在根本性的偏差，称为**对齐问题（Alignment Problem）**。
```
预训练目标：P(token_t | token_1, ..., token_{t-1})
人类真实目标：生成有价值、安全、符合价值观的内容
                        ↕ 存在 Gap
```
简单的监督微调（SFT）虽然能改善输出风格，但：
- **人工标注成本极高**，难以覆盖所有场景
- 模型容易**过拟合标注数据**，泛化性差
- 无法有效表达人类对"好坏"的**相对偏好**
RLHF 的核心思想是：**用人类偏好信号训练一个代理奖励函数，再用 RL 优化语言模型**。

## 三、Step 1 & 2：预训练 + 监督微调
### 预训练（Pre-training）
在海量无标注文本上，以自回归方式预测下一个 token：
$$\mathcal{L}_{PT} = -\sum_{t=1}^{T} \log P_\theta(x_t \mid x_1, \ldots, x_{t-1})$$

得到基础模型 $\pi^{PT}$，具备语言能力但不懂"如何遵循指令"。
### 监督微调（Supervised Fine-Tuning, SFT）
收集高质量的**（指令，理想回答）**对，进行有监督训练：
$$\mathcal{L}_{SFT} = -\sum_{t=1}^{T} \log P_\theta(y_t \mid x, y_1, \ldots, y_{t-1})$$
得到 SFT 模型 $\pi^{SFT}$，是后续步骤的**起点与参考基准**。

---
## 四、Step 3：奖励模型训练（核心）
### 4.1 数据收集：偏好比较
不直接让人类打绝对分数（主观性太强），而是让标注者对**同一问题的多个回答进行排序**：
```
问题 x：如何缓解压力？

回答 A：建议你……（较好）
回答 B：你可以……（较差）
回答 C：……（最差）

标注者排序：A > B > C
→ 产生偏好对：(A > B), (A > C), (B > C)
```
### 4.2 Bradley-Terry 偏好模型
用 **Bradley-Terry 模型**对人类偏好建模。假设每个回答 $y$ 有一个潜在的"质量分" $r(x, y)$，则人类选择 $y_w$（winner）优于 $y_l$（loser）的概率为：
$$P(y_w \succ y_l \mid x) = \sigma(r(x, y_w) - r(x, y_l)) = \frac{e^{r(x,y_w)}}{e^{r(x,y_w)} + e^{r(x,y_l)}}$$
其中 $\sigma$ 是 Sigmoid 函数。
### 4.3 奖励模型的训练目标
将语言模型最后一层替换为线性层输出标量分数，训练目标为**最大化人类偏好对数似然**（即最小化负对数似然）：
$$\mathcal{L}_{RM} = -\mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}} \left[ \log \sigma\left( r_\phi(x, y_w) - r_\phi(x, y_l) \right) \right]$$
**直觉理解**：让"好回答"的分数尽可能高于"差回答"的分数。
### 4.4 奖励模型的架构
```
输入：[x; y]（问题 + 回答拼接）
  ↓
Transformer Layers（从 SFT 模型初始化）
  ↓
最后一个 token 的隐藏状态
  ↓
Linear(d_model → 1)
  ↓
输出：标量分数 r
```
> **关键设计**：奖励模型通常从 SFT 模型初始化，这样它已经具备丰富的语言理解能力。

## 五、Step 4：PPO 强化学习优化
### 5.1 问题定义（MDP 视角）
将文本生成建模为一个**马尔可夫决策过程（MDP）**：

|MDP 元素|对应含义|
|---|---|
|**状态 $s_t$**|当前已生成的 token 序列|
|**动作 $a_t$**|从词表中选择下一个 token|
|**策略 $\pi_\theta$**|语言模型本身|
|**奖励 $r$**|仅在序列结束时由 RM 给出|
|**环境**|自回归解码过程|
目标：找到策略 $\pi_\theta$ 最大化期望奖励：

$$\max_\theta \mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\theta(\cdot|x)} [r_\phi(x, y)]$$
### 5.2 KL 散度惩罚：防止模型"跑偏"
纯粹最大化奖励会导致**奖励欺骗（Reward Hacking）**：模型可能生成语义奇异但奖励模型打高分的文本。因此加入 **KL 散度惩罚**：
$$\mathcal{R}(x, y) = r_\phi(x, y) - \beta \cdot \underbrace{D_{KL}(\pi_\theta(\cdot|x) | \pi^{SFT}(\cdot|x))}_{\text{防止偏离参考模型}}$$
$$D_{KL} = \sum_t \log \frac{\pi_\theta(a_t|s_t)}{\pi^{SFT}(a_t|s_t)}$$
- $\beta$ 越大：越保守，越接近 SFT 模型
- $\beta$ 越小：越激进，更敢于探索高奖励区域
有时也加入预训练语言模型损失，防止模型遗忘基础语言能力：
$$\mathcal{L}_{total} = -\mathcal{R}(x,y) + \gamma \mathcal{L}_{PT}$$
### 5.3 PPO 算法详解
PPO（Proximal Policy Optimization）是 RLHF 中最常用的算法，核心思想是**限制每次策略更新的幅度**。
**5.3.1 优势函数估计（GAE）**
用 Critic 网络（价值函数 $V_\psi$）估计广义优势函数（GAE）：
$$\hat{A}_t = \sum_{l=0}^{\infty} (\gamma \lambda)^l \delta_{t+l}, \quad \delta_t = r_t + \gamma V_\psi(s_{t+1}) - V_\psi(s_t)$$
**5.3.2 PPO-Clip 目标函数**
$$\mathcal{L}_{PPO} = \mathbb{E}_t \left[ \min\left( \rho_t \hat{A}_t,\ \text{clip}(\rho_t, 1-\epsilon, 1+\epsilon) \hat{A}_t \right) \right]$$
其中 $\rho_t = \dfrac{\pi_\theta(a_t|s_t)}{\pi_{old}(a_t|s_t)}$ 是重要性权重（新旧策略的概率比）。
**直觉理解**：
- 如果新策略比旧策略好（$A > 0$），允许提升但不超过 $1+\epsilon$
- 如果新策略比旧策略差（$A < 0$），允许下调但不低于 $1-\epsilon$
- 本质是**信任域约束**的近似实现
**5.3.3 完整 PPO 训练循环**
```
for each iteration:
    1. 用当前策略 π_θ 采样一批回答
    2. 用奖励模型 r_φ 打分 + KL 惩罚 → 计算 R(x,y)
    3. 用 Critic V_ψ 计算优势估计 Â
    4. 多个 mini-epoch 更新：
       - Actor (π_θ) → PPO-Clip Loss
       - Critic (V_ψ) → MSE Loss
    5. 更新参考策略 π_old ← π_θ
```

## 六、训练稳定性技巧

|技巧|说明|
|---|---|
|**奖励归一化**|对 batch 内奖励做 z-score 归一化，防止梯度爆炸|
|**KL 自适应系数**|动态调整 $\beta$，让 KL 散度维持在目标范围|
|**值函数预训练**|先用 SFT 数据预热 Critic，减少训练初期的方差|
|**分离参考模型**|参考模型 $\pi^{SFT}$ 冻结，不参与梯度更新|
|**梯度裁剪**|防止策略网络梯度爆炸|

## 七、RLHF 的局限与改进方向
### 局限性
```
奖励欺骗         → 模型发现奖励模型的漏洞
偏好噪声         → 标注者意见不一致
分布偏移         → RM 在 RL 探索的新分布上可能失效
成本高昂         → PPO 需要同时维护 4 个模型（Actor/Critic/RM/Ref）
稀疏奖励         → 只在序列末尾给奖励，信用分配困难
```
### 主要改进方向
**① DPO（Direct Preference Optimization，2023）** 绕过显式奖励模型，直接从偏好数据优化策略：

$$\mathcal{L}_{DPO} = -\mathbb{E} \left[ \log \sigma \left( \beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)} \right) \right]$$

优点：无需单独训练 RM，更稳定；缺点：离线方法，无法在线探索。
**② GRPO（Group Relative Policy Optimization，DeepSeek-R1）** 去掉 Critic 网络，用组内相对奖励替代优势估计，大幅降低显存占用：
$$\hat{A}_i = \frac{r_i - \text{mean}(r_1,...,r_G)}{\text{std}(r_1,...,r_G)}$$
**③ Constitutional AI（Anthropic）** 用 AI 自我评判替代人工标注，通过预设"宪法原则"指导模型自我改进。
**④ 过程奖励模型（PRM）** 对推理的每一步打分，而非只看最终结果，更适合数学和逻辑推理任务。

## 八、总结
```
RLHF 的本质：
将"人类偏好"这一难以形式化的目标，
通过 Bradley-Terry 模型转化为可微的奖励函数，
再用 PPO 算法在语言生成的巨大动作空间中优化策略，
同时用 KL 惩罚保证模型不偏离语言能力基准。

核心公式链：
人类偏好数据 → RM Loss → 奖励模型 r_φ
                                  ↓
SFT 模型 π_SFT → PPO + KL 惩罚 → 对齐后的策略 π_θ
```
RLHF 的成功证明了**将人类判断转化为优化信号**的可行性，但其高成本和不稳定性也催生了 DPO、GRPO 等更高效的替代方案。理解 RLHF 是理解现代大模型对齐技术的基石。
