# Diffusion LLM：扩散式大语言模型
## 一、背景：自回归模型的局限
主流大语言模型（如 GPT、LLaMA）采用**自回归（Autoregressive, AR）范式**，逐 token 从左至右生成文本：
$$P(x) = \prod_{t=1}^{T} P(x_t \mid x_{<t})$$
这带来几个固有缺陷：
- **串行生成**：每步依赖上一步输出，无法并行，推理延迟随序列长度线性增长
- **暴露偏差（Exposure Bias）**：训练时用 ground truth，推理时用自身预测，存在分布偏移
- **单向上下文**：生成时无法利用未来 token 的信息
---
## 二、核心思想：将扩散过程引入离散文本空间
扩散模型（Diffusion Model）在连续域（图像、音频）已取得巨大成功，核心思路是：
> 学习逆转一个将数据逐步破坏为噪声的马尔可夫过程。
将其迁移到**离散 token 序列**是 Diffusion LLM 的核心挑战。
### 2.1 前向过程（Forward Process）
将原始 token 序列 $x_0$ 逐步加噪，直至变为纯噪声分布（如均匀分布或 `[MASK]`）：
$$q(x_t \mid x_{t-1})$$
两种主流加噪方式：

|类型|代表工作|机制|
|---|---|---|
|**Absorbing Diffusion**|D3PM, MDLM|token 以一定概率被替换为 `[MASK]`|
|**Uniform Diffusion**|D3PM|token 以一定概率被随机替换为词表中任意 token|

以 Absorbing 为例，在时间步 $t$，每个 token 独立地以概率 $\bar{\alpha}_t$ 保持原值，以 $1-\bar{\alpha}_t$ 被替换为 `[MASK]`。
### 2.2 逆向过程（Reverse Process）
训练神经网络（通常为 Transformer）学习去噪：
$$p_\theta(x_{t-1} \mid x_t)$$
模型在每步预测被 mask/损坏的 token 的原始值，逐步从噪声中恢复出完整序列。推理时，**所有 token 并行更新**，经过 $T$ 步迭代得到最终结果。

---
## 三、关键技术突破
### 3.1 训练目标
主流目标为**Evidence Lower Bound (ELBO)**的离散变体：
$$\mathcal{L} = \mathbb{E}_q \left[ \sum_{t=1}^{T} D_{\text{KL}}(q(x_{t-1}|x_t, x_0) | p_\theta(x_{t-1}|x_t)) \right]$$
MDLM（Masked Diffusion Language Model）将其简化为一个加权的**交叉熵损失**，大幅提升训练稳定性。
### 3.2 架构选择
Diffusion LLM 的骨干网络通常为**双向 Transformer**（非 causal），因为：
- 逆向去噪需要利用序列的**全局上下文**
- 不再受因果注意力掩码约束
这是与 AR 模型最重要的架构区别之一。
### 3.3 条件生成
通过 Classifier-Free Guidance（CFG）或 prefix conditioning 实现条件控制：
$$\tilde{p}_\theta(x_0 \mid c) \propto p_\theta(x_0 \mid c)^\gamma \cdot p_\theta(x_0)^{1-\gamma}$$
其中 $\gamma$ 为引导强度。

## 四、代表性工作演进
```
D3PM (Austin et al., 2021)
  └─ 首次系统建立离散扩散框架

Multinomial Diffusion (Hoogeboom et al., 2021)
  └─ 类别分布上的扩散

MDLM (Sahoo et al., 2024)
  └─ 简化 ELBO，absorbing diffusion + 高效训练

SEDD (Lou et al., 2024)
  └─ Score Entropy Discrete Diffusion，引入得分匹配

LLaDA (Meta, 2025)
  └─ 大规模 Masked Diffusion LLM，首次在指令跟随上
     与同量级 AR 模型竞争

Mercury (Inception Labs, 2025)
  └─ 首个商业级 Diffusion LLM，主打推理速度
```

## 五、与自回归模型的系统对比

|维度|自回归 LLM|Diffusion LLM|
|---|---|---|
|**生成方式**|串行，逐 token|并行，多步迭代|
|**上下文**|单向（因果）|双向（全局）|
|**推理延迟**|$O(T)$ 串行步|可控迭代步数（通常 <100）|
|**吞吐量**|受 KV-Cache 限制|天然并行，GPU 利用率高|
|**可控生成**|较难（需后处理）|原生支持（infilling, constrained）|
|**训练稳定性**|成熟|仍在优化中|
|**生成质量**|当前更优|快速追赶中|

## 六、核心优势与现存挑战
**优势：**
- **推理加速**：并行解码使吞吐量可达 AR 模型数倍
- **双向建模**：天然支持填空（infilling）、受约束生成
- **迭代精化**：可在算力预算内调整质量/速度权衡
**挑战：**
- **困惑度（Perplexity）仍落后**：在标准语言建模 benchmark 上与顶级 AR 模型有差距
- **长序列扩展**：噪声调度设计对长文本仍不成熟
- **采样步数权衡**：步数少则质量下降，步数多则失去速度优势
- **理论理解不足**：离散扩散的理论基础不如连续域完善
---
## 七、前沿方向
- **连续潜空间扩散**：在 token embedding 的连续空间做扩散（如 Diffusion-LM），绕开离散化难题
- **与 AR 的混合架构**：兼顾两者优势
- **步数蒸馏**：借鉴 Consistency Model 思想，将多步推理压缩至少步
- **scaling law 研究**：探索 Diffusion LLM 的规模化规律是否与 AR 一致
---
Diffusion LLM 代表了语言生成范式的一次重要探索。尽管当前在生成质量上尚未全面超越自回归模型，但其在**推理效率**和**双向生成能力**上的结构性优势，使其成为下一代语言模型架构的重要竞争者。
