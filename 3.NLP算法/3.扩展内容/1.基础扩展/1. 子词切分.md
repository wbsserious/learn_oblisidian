**定义：** 子词切分是一种介于**单词**与**字符**之间的文本切分方式，将文本切分为**词缀、词根、常用组合**等子词单元，是深度学习 NLP（尤其是 Transformer 类模型）的标准分词方式。
# 核心思想
- 英文：把长词拆成有意义的片段
    - `unhappiness → un + happiness`
    - `playing → play + ing`
- 中文：按高频组合切分，不依赖人工词典
- 既不像字符那样太碎，也不像单词那样无法处理生词
# 常见方法
- **BPE**：GPT，RoBERTa常用
- **WordPiece**：BERT 常用
- **SentencePiece**：多语言、中日韩统一切分
# 中文子词切分的特点
中文没有空格，不能像英文一样先按照单词进行切分。
现代模型直接以“字”为基础进行切分，再合成子词。
不在需要传统中文切词（jieba切词）
**中文里最主流、最常用的子词切分：WordPiece 和 BPE。**

# 中文子词切分概念
中文子词切分是现代大模型（如 BERT、GPT、ERNIE）的核心预处理技术，它**不依赖传统词典分词**，而是以**单字为原子单元**，通过算法动态合并高频组合，形成兼顾语义与效率的子词序列，完美解决中文无天然词边界、OOV（未登录词）多、词表膨胀等问题。

### 一、中文子词切分的核心特点

- **原子起点**：所有算法均以**单个汉字**为最小初始单元（如 “我”“爱”“自”“然”）。
- **动态合并**：基于语料统计，迭代合并高频共现的字对 / 子词对，生成复合子词（如 “自然”“语言”）。
- **词表可控**：通过设定词汇表大小，平衡覆盖度与计算效率。
- **OOV 友好**：罕见词 / 新词自动拆分为已知子词，避免 [UNK]。
- **无歧义**：无需处理传统分词的歧义（如 “发展中国家”）。

### 二、主流算法在中文中的应用

#### 1. BPE（Byte-Pair Encoding，字节对编码）

- **原理**：从单字开始，**每次合并频率最高的相邻字对**，直到达到预设词表大小。
- **中文示例**：
    
    1. 初始词表：`{我, 爱, 自, 然, 语, 言, 处, 理}`
    2. 统计高频对：`自+然`、`语+言` 频率最高
    3. 合并：词表加入`自然`、`语言`
    4. 重复迭代，最终词表包含单字 + 高频复合子词
    
- **代表模型**：GPT 系列、LLaMA、BART
- **中文优势**：简单高效，适合生成式模型；**缺点**：可能拆分语义完整的词

#### 2. WordPiece（BERT 系主流）

- **原理**：类似 BPE，但**合并标准是最大化语言模型似然增益**（即`P(ab)/(P(a)×P(b))`最大），更关注语义连贯性。
- **中文示例**：
    
    - 输入：`我爱自然语言处理`
    - 切分：`我 爱 自然 语言 处 理`（高频词合并，低频字保留）
    - 标记：子词前缀加`##`（如`##然`），区分是否为词首
    
- **代表模型**：BERT-base-chinese、ERNIE、DistilBERT
- **中文优势**：语义更完整，适合理解式任务；**缺点**：计算略复杂

#### 3. SentencePiece（多语言 / 字节级）

- **原理**：**不依赖空格 / 分词**，直接将文本视为 Unicode 字节流，在字节 / 字符级做 BPE/Unigram，支持任意语言（含中文、 emoji、代码）。
- **中文特点**：
    
    - 可配置为`char`（单字）或`bpe`模式
    - 输出可逆，支持端到端训练
    - 适合多语言模型（如 mBERT、T5、BLOOM）
    
- **代表模型**：T5、mBERT、ALBERT、部分多语言 LLM
