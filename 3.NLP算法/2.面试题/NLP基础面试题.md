# Q1: 什么是自然语言处理（NLP）？它的核心目标是什么？

自然语言处理（Natural Language Processing, NLP）是计算机科学与语言学的交叉领域，**核心目标是让机器能够理解和生成自然语言**，实现人与计算机之间通过语言进行自然交互。

# Q2: NLP 与结构化数据、图像数据相比，主要难点是什么？

自然语言数据（如汉语、英语等）不像结构化数据或图像数据可以方便地进行数值化表示，**文本是非结构化的、具有歧义性和上下文依赖性**，需要特殊的编码和建模方法。

# Q3: 请列举至少 5 种 NLP 的主要任务，并简要说明。

1. **语音识别：** 将语音信号转化为文本
2. **机器翻译：** 自动将一种语言翻译为另一种语言
3. **命名实体识别（NER）：** 识别文本中的人名、地名、组织名等实体
4. **情感分析：** 判断文本的情感倾向（正面/负面/中性）
5. **对话系统：** 使计算机能与人类进行自然流畅的对话（如智能助手）
6. **自动摘要：** 从长文本中提取关键信息生成简洁摘要

# Q4: 中文分词与英文分词的主要区别是什么？为什么中文分词更具挑战性？

英文单词之间**以空格作为自然分界符**，分词相对简单；而中文只有字、句、段有明显分界符，**词没有形式上的分界符**，需要通过算法识别词的边界，因此更具挑战性。

# Q5: Jieba 分词支持哪三种模式？各适用于什么场景？

- **精确模式：** 试图将句子最精确地切开，适合**文本分析**场景
- **全模式：** 扫描出所有可能成词的词语，速度快但存在歧义，适合**快速扫描**
- **搜索引擎模式：** 在精确模式基础上对长词再次切分，提高召回率，适合**搜索引擎分词**

# Q6: NER 在实际的 NLP 系统中有哪些典型应用？

- **信息抽取：** 从海量文本中自动提取结构化实体信息
- **问答系统：** 准确理解用户提问中的关键实体
- **文本理解：** 识别文本关键信息，支持语义分析和事件抽取
- **知识图谱构建：** 抽取实体作为图谱节点

# Q7: 在实际项目中，如果 Jieba 对专业领域词汇（如"黑马程序员"）分词效果不佳，你会如何解决？

1. **加载自定义词典：** 将专业词汇加入用户词典，指定词频和词性
2. **调整词典词频：** 提高专业词汇的词频权重，使其优先被识别为一个整体
3. **使用领域语料训练：** 如有条件，使用领域特定语料重新训练分词模型

# Q8: 什么是文本张量表示？为什么需要将文本转换为张量形式？

文本张量表示是将文本使用张量（矩阵）进行数值化表示的方法，一般将词汇表示成向量（词向量），再由词向量组成矩阵。
**核心作用：** 计算机无法直接理解人类语言，张量表示作为**桥梁**将文本转换为数值形式，使其能够作为机器学习/深度学习模型的输入。

# Q9: One-Hot 编码的基本原理是什么？有什么主要缺陷？

原理：每个词表示为长度为词汇表大小的稀疏向量，只有对应位置为 1，其余为 0。 
**主要缺陷：**
- **维度灾难：** 词汇表越大，向量维度越高（10万词表=10万维向量）
- **语义鸿沟：** 任意两个词的向量正交，无法表达词与词之间的语义关系（如"cat"和"dog"的向量完全不同）
- **内存浪费：** 高维稀疏向量占据大量存储空间

# Q10: Word2Vec 的核心思想是什么？它如何解决 One-Hot 的缺陷？

 核心思想：利用**上下文信息**学习词的分布式表示，通过神经网络训练将词映射为低维稠密向量。

 解决 One-Hot 缺陷：

- **降维：** 通常 50-300 维，远低于 one-hot 的词汇表维度
- **语义捕捉：** 语义相似的词在向量空间中距离近（如通过向量运算 "king" - "man" + "woman" ≈ "queen"）

# Q11: Word2Vec 包含哪两种训练模式？它们的主要区别是什么？

| 模式            | 输入    | 输出      | 适用场景             |

# Q12: 为什么 Embedding 层的词向量能够捕捉语义信息？

在神经网络训练过程中，模型通过**反向传播**不断调整 Embedding 层的参数。语义相似的词在相似上下文中出现，会获得相似的梯度更新，最终在向量空间中聚集在一起。因此训练完成后，语义相关的词具有相近的向量表示。

# Q13: One-Hot、Word2Vec、神经网络 Embedding 层三种表示方法有何区别？

| 特性         | One-Hot            | Word2Vec       | Embedding 层       |
| ------------ | ------------------ | -------------- | ------------------ |
| **维度**     | 高维（词汇表大小） | 低维（50-300） | 低维（可自定义）   |
| **稀疏性**   | 稀疏               | 稠密           | 稠密               |
| **语义关系** | 无                 | 有             | 有（随任务学习）   |
| **训练方式** | 无需训练           | 无监督预训练   | 与下游任务联合训练 |
| **适用场景** | 简单 baseline      | 通用词向量     | 特定任务优化       |

# Q14: 什么是文本数据分析？它在 NLP 项目中的主要作用是什么？

文本数据分析是从非结构化文本数据中提取有价值信息、模式和见解的过程。 

**主要作用：**
- **理解数据语料：** 掌握数据分布特征和质量问题
- **发现潜在问题：** 快速检查标签不平衡、异常样本等问题
- **指导超参数选择：** 为模型输入长度、批次大小等提供数据依据

# Q15: 在深度学习模型训练前，为什么要进行标签数量分布分析？

- **检测类别不平衡：** 避免模型倾向于预测样本多的类别
- **设定评估基线：** 若正负样本比例 1:1，ACC 基线约为 50%；若严重不平衡，需调整评估指标（如 F1-score）
- **指导数据策略：** 决定是否需要数据增强、重采样或调整类别权重

# Q16: 分析句子长度分布对模型训练有什么实际指导意义？

- **设置模型输入长度：** 根据分布范围确定 `max_length`（如大部分句子在 20-250 之间，可设为 256）
- **决定预处理策略：** 过长句子截断、过短句子填充
- **优化计算资源：** 避免过度填充造成内存浪费，或截断导致信息丢失
- **发现异常值：** 识别超长样本（如近 3500 字符）进行人工审查

# Q17: 什么是 n-gram 特征？常用的 n-gram 有哪些？

n-gram 特征是将文本中相邻的 n 个词（或字符）组合起来形成的新特征表示。
常用的有：
- **bi-gram（2-gram）：** 相邻 2 个词组合，如"是谁"+"敲动"
- **tri-gram（3-gram）：** 相邻 3 个词组合
例如：`["是谁", "敲动", "我心"]` 的 bi-gram 特征为 `{(是谁,敲动), (敲动,我心)}`

# Q18: 为什么需要引入 n-gram 特征？举例说明其与 uni-gram 的区别。

n-gram 能**捕捉局部上下文信息**，而 uni-gram 孤立看待每个词。

**关键区别示例：**

- uni-gram 只看到 "not" 和 "happy"，可能误判为正面
- bi-gram "not happy" 整体作为特征，能正确识别负面情感

**优势：** 无需预训练、简单高效、直接反映词序关系。

# Q19：文本长度规范的两种核心操作是什么？请说明各自策略。

| 操作                  | 策略                                                      | 适用场景                        |
| --------------------- | --------------------------------------------------------- | ------------------------------- |
| **截断 (Truncation)** | 前截断（`pre`）：丢弃开头；后截断（`post`）：丢弃结尾     | 文本过长，保留尾部/头部关键信息 |
| **填充 (Padding)**    | 前填充（`pre`）：在开头补 0；后填充（`post`）：在结尾补 0 | 文本过短，保持固定长度          |

# Q20: 如何确定文本长度规范中的 max_length 值？

基于**句子长度分布分析**：

1. 统计语料中所有句子长度
2. 绘制分布图（柱状图/密度曲线）
3. 选择覆盖 **90%-95%** 样本的最短长度
4. 权衡：过长 → 计算浪费；过短 → 信息丢失

例如：若 90% 句子长度 ≤ 250，则 `max_length=256`（取 2 的幂次利于 GPU 计算）

# Q21: 为什么 n-gram 特征在深度学习时代仍有价值？

- **传统模型（LR、SVM）：** n-gram 是核心特征，直接提供判别信息
- **深度学习：** 虽 CNN/Transformer 能自动学习 n-gram 模式，但**显式加入 n-gram 特征**仍能：
    - 加速收敛（提供先验知识）
    - 提升小样本场景表现
    - 增强可解释性

# Q22: 什么是回译数据增强法？它的核心原理是什么？

回译数据增强是将文本先翻译成另一种语言（中间语言），再翻译回原语言，从而获得**语义相同但句式不同**的新样本。

**核心原理：** 利用不同语言的语法结构差异，同一语义在翻译过程中会产生句式变化，增加数据多样性。

# Q23: 什么是 RNN（循环神经网络）？它相比传统神经网络的核心优势是什么？

RNN 是专门用于**处理序列数据**的神经网络架构，通过**循环连接**使隐藏层上一时间步的输出能作为当前时间步输入的一部分。

**核心优势：** 能够**捕捉序列中的时间依赖关系**，记住过去信息并影响当前决策，适用于文本、语音、时间序列等有序数据。

# Q24：请解释 RNN 中的"隐藏状态（Hidden State）"和"时间步（Time Step）"概念。

- **隐藏状态（ht ）：** RNN 的"记忆"，包含过去所有时间步的信息，由当前输入和上一隐藏状态共同计算得到
- **时间步：** 序列中的单个元素（如一个单词、一帧音频），RNN 按时间步顺序逐个处理

# Q25: 传统 RNN 的主要优点是什么？适用于哪些场景？

**优点：**
- 结构简单，易于理解和实现
- 参数总量少，计算资源要求低
- 能处理可变长度序列
- 在**短序列任务**上表现优异

**适用场景：** 简单文本分类、短句情感分析、计算资源受限的环境、作为学习 LSTM/GRU 的基础。

# Q26: 为什么 RNN 无法并行计算？这对训练效率有什么影响？

RNN 的**循环结构具有时间依赖性**，当前时间步的计算必须等待上一时间步的隐藏状态输出，形成顺序依赖链。

**影响：**

- 无法利用 GPU 批量并行加速
- 训练速度受序列长度线性影响
- 长序列训练耗时显著增加

# Q27: LSTM 相比传统 RNN 的核心改进是什么？它通过什么机制解决了梯度消失问题？

LSTM 引入**门控机制**（遗忘门、输入门、输出门）和**细胞状态（Cell State）**，通过门控制信息流动。

**解决梯度消失的关键：** 细胞状态更新是**线性关系**（$$C_t=f_t⋅C_{t−1}+i_t⋅\tilde{C}_t $$），梯度通过细胞状态传播时保持线性，避免了传统 RNN 的连乘衰减

# Q28: 请分别解释 LSTM 的三个门控及其作用。

| 门控             | 作用                     | 公式                                     |
| ---------------- | ------------------------ | ---------------------------------------- |
| **遗忘门** $f_t$ | 决定丢弃多少历史信息     | $\sigma(W_f \cdot [h_{t-1}, x_t] + b_f)$ |
| **输入门** $i_t$ | 决定存储多少新信息       | $\sigma(W_i \cdot [h_{t-1}, x_t] + b_i)$ |
| **输出门** $o_t$ | 决定输出多少细胞状态信息 | $\sigma(W_o \cdot [h_{t-1}, x_t] + b_o)$ |

# Q29: LSTM 的细胞状态（Cell State）和隐藏状态（Hidden State）有什么区别？

- **细胞状态 Ct ：** 贯穿时间步的"传送带"，携带**长期记忆**，更新缓慢（线性变换），信息不易丢失
- **隐藏状态 ht ：** 当前时间步的**输出**，由输出门过滤后的细胞状态经过 tanh 激活得到，作为下一时间步输入的一部分

# Q30: GRU 相比 LSTM 做了哪些简化？它的两个门控分别是什么？

 GRU 将 LSTM 的**三个门简化为两个门**，合并细胞状态和隐藏状态：
- **更新门 zt ：** 控制保留多少旧状态、更新多少新状态（融合遗忘门+输入门）
- **重置门 rt ：** 控制忽略多少历史信息，决定候选状态的计算
# Q31: 注意力机制最初是为了解决 Seq2Seq 的什么问题而提出的？

1. **信息瓶颈：** 编码器将长序列压缩为固定长度上下文向量，导致信息丢失
2. **长序列性能下降：** 长句子翻译时模型准确率严重下降
注意力机制通过**动态关注**输入序列的不同部分，避免信息瓶颈，提升长序列处理能力。

# Q32: 软注意力（Soft Attention）和硬注意力（Hard Attention）有什么区别？

| 特性       | Soft Attention                   | Hard Attention |
| -------- | -------------------------------- | -------------- |
| **权重分配** | 0-1 之间的连续值                       | 非 0 即 1（选择或舍弃） |
| **计算方式** | 加权求和所有输入                         | 只选择得分最高的或随机采样  |
| **可导性**  | 可导，可用反向传播                        | 不可导，需强化学习训练    |
| **应用**   | 主流方法（Transformer、LSTM+Attention） | 较少使用           |
|          |                                  |                |
# Q33: 自注意力（Self-Attention）与一般注意力有什么区别？有什么优势？

**区别：** Self-Attention 的 **Query = Key = Value**，都是输入序列本身，计算序列内部元素之间的关联。
**优势：**
- **长距离依赖：** 任意两个 token 直接计算关联，无需经过中间步骤（RNN 需要逐步传递）
- **并行计算：** 不依赖时序，可一次性计算整个序列的注意力
- **可解释性：** 注意力权重直接显示词与词之间的关系

# Q34: 缩放点积注意力为什么要除以根号dk ？

当 dk 较大时，点积值会很大，导致 Softmax 梯度极小（饱和区），难以训练。除以dk将值缩放到合理范围，保持梯度稳定。

# Q35: 在 Seq2Seq 模型中，编码器端和解码器端的注意力机制分别起什么作用？

| 位置       | 注意力类型                                      | 作用                               |
| -------- | ------------------------------------------ | -------------------------------- |
| **编码器端** | Self-Attention                             | 特征提取，捕捉输入序列内部依赖关系，增强表示           |
| **解码器端** | Encoder-Decoder Attention + Self-Attention | 关注编码器输出，动态选择相关信息生成当前词；同时关注已生成的输出 |
|          |                                            |                                  |
**关键区别：** 编码器端 Q/K/V 都来自输入；解码器端 Q 来自解码器状态，K/V 来自编码器输出

# Q36: 请说出注意力机制的三步计算过程。

1. **计算注意力分数：** Query 与 Key 计算相似度（点积/加性/拼接线性变换）
2. **Softmax 归一化：** 将分数转换为概率分布（权重和为 1）
3. **加权求和：** 权重与 Value 相乘求和，得到注意力输出（上下文向量）

# Q37: Transformer 相比 RNN/LSTM 的核心优势是什么？它如何解决长距离依赖问题？

核心优势是**完全基于自注意力机制，摒弃了循环结构**，实现了：
- **并行计算：** 可同时处理整个序列，大幅提升训练效率
- **长距离依赖：** 自注意力直接计算任意两个位置的关系，距离为 O(1)，而 RNN 为 O(n)
- **解决方式：** 通过缩放点积注意力 **Attention** (*Q*,*K*,*V*)=***softmax***(***dk，Q，K，T)V***  直接建模全局关系

# q38：为什么 Transformer 需要位置编码？它是如何实现的？

因为 Transformer **没有循环或卷积结构**，无法感知序列顺序。通过正弦余弦编码实现（周期性、平滑性、有界性，且位置间存在线性关系，便于学习相对位置。）

# Q39: Transformer 编码器和解码器的主要区别是什么？

| 组件           | 编码器              | 解码器                                |     |
| ------------ | ---------------- | ---------------------------------- | --- |
| **注意力类型**    | 多头自注意力（双向）       | 掩码自注意力（单向）+ 编码器-解码器注意力             |     |
| **层数**       | 2 个子层（自注意力 + 前馈） | 3 个子层（掩码自注意力 + 交叉注意力 + 前馈）         |     |
| **掩码**       | 仅需 Padding Mask  | 需 Padding Mask + Look-Ahead Mask   |     |
| **Q/K/V 来源** | 都来自输入序列          | 自注意力来自输出序列，交叉注意力 Q 来自解码器，K/V 来自编码器 |     |

# Q40: 什么是多头注意力机制？为什么要用多个头？

将 Q/K/V 通过线性变换投影到h个子空间，并行计算注意力，再拼接结果。

**多头作用：**
- **多视角学习：** 不同头关注不同特征（如语法、语义、指代）
- **增强表达：** $MultiHead(Q,K,V)=Concat(head_1,...,head_h)W^O$ 
- **计算效率：** 每个头维度 `d_k=dmodel/h` ，总计算量与单头相近

# Q41: Transformer 中的掩码（Mask）有哪些类型？各自作用是什么？

| 掩码类型                        | 作用                                            | 应用场景       |
| ------------------------------- | ----------------------------------------------- | -------------- |
| **Padding Mask**                | 屏蔽填充位置（`<PAD>`），避免注意力关注无效位置 | 编码器、解码器 |
| **Look-Ahead Mask（因果掩码）** | 屏蔽未来位置，确保自回归生成                    | 解码器自注意力 |

# Q42: 残差连接（Residual Connection）和层归一化（LayerNorm）在 Transformer 中为什么重要？

- **残差连接：** $Output=LayerNorm(x+Sublayer(x))$ 
    - 缓解梯度消失，使深层网络可训练（Transformer 通常 6-12 层）
    - 保留原始信息，加速收敛
- **层归一化：** 对每个样本的所有特征做归一化
    - 稳定训练过程，减少内部协变量偏移
    - 相比 BatchNorm，更适合变长序列和较小 batch

# Q43: 在机器翻译任务中，Transformer 训练时和推理时的解码器输入有何不同？

| 阶段     | 输入                      | 说明                         |
| ------ | ----------------------- | -------------------------- |
| **训练** | 真实目标序列（Teacher Forcing） | 并行计算，使用完整的 Look-Ahead Mask |
| **推理** | 已生成的序列（自回归）             | 逐个生成，每次将新词加入输入             |

**关键区别：** 训练时可并行，推理时必须自回归，因为不知道未来的输出。

# Q44: 为什么 Transformer 适合构建预训练语言模型（如 BERT、GPT）？

- **双向/单向灵活：** 编码器可双向建模（BERT），解码器可单向生成（GPT）
- **可扩展性：** 通过增加层数、头数、维度可持续提升能力
- **迁移学习友好：** 预训练的注意力权重包含丰富语义知识，易迁移到下游任务
- **并行训练：** 相比 RNN，可高效处理大规模语料

# Q45: FastText 相比 Word2Vec 的核心改进是什么？它如何解决未登录词（OOV）问题？

**核心改进：引入子词信息（Subword Information）**

**解决 OOV 方式：** 将单词拆分为字符级 n-grams（如 "apple" → `<ap, app, ppl, ple, le>`），每个 n-gram 学习独立向量，单词向量为其所有 n-gram 向量的平均。即使未登录词，也可通过子词组合生成向量。

# Q46: FastText 有哪些优缺点？适用于什么场景？
**优点：**
- 训练速度极快（层次 Softmax + 负采样）
- 处理 OOV 能力强（子词信息）
- 适合大规模文本分类
**缺点：**
- 无法捕捉词序信息（需配合 n-gram 特征弥补）
- 短文本效果不如深度学习模型
- 需要大量数据训练高质量模型
**适用场景：** 大规模文本分类（如垃圾邮件识别、情感分析）、资源受限环境、需要快速 baseline。
# Q47: 什么是迁移学习？在深度学习 NLP 中，预训练-微调范式为什么有效？
**迁移学习：** 将源任务（大数据集）学到的知识迁移到目标任务（小数据集），减少目标数据需求，加速训练，提升性能。
**预训练-微调有效原因：**
1. **通用特征学习：** 预训练模型在大规模语料上学习语言通用规律（语法、语义）
2. **良好初始化：** 避免随机初始化，提供高质量起点
3. **正则化效果：** 预训练权重作为先验，防止过拟合
4. **数据效率：** 少量标注数据即可达到较好效果
# Q48: 预训练模型和微调（Fine-tuning）有什么区别？主流的微调策略有哪些？

| 对比项       | 预训练模型       | 微调             |
| ------------ | ---------------- | ---------------- |
| **数据**     | 大规模无标注数据 | 任务相关标注数据 |
| **目标**     | 学习通用表示     | 适应特定任务     |
| **参数更新** | 全量更新         | 部分或全量更新   |
| **计算成本** | 高（只需一次）   | 相对较低         |

**主流微调策略：**
1. **全量微调：** 更新所有参数，效果最佳但易过拟合
2. **冻结底层：** 冻结预训练层，只训练顶层/任务头，节省计算
3. **逐步解冻：** 先训练顶层，再逐步解冻下层
4. **差异学习率：** 底层用较小学习率，顶层用较大学习率

# Q49: 为什么 BERT 等预训练模型在下游任务上通常只需要少量标注数据？

- **预训练已学习语言知识：** 在大规模语料上通过 MLM 和 NSP 任务学习深层语义
- **特征复用：** 底层编码通用语法，顶层适配特定任务，只需微调顶层即可
- **注意力机制：** 自注意力已建模全局依赖，下游任务只需调整注意力分布
- **参数高效：** 相比从头训练，微调只需调整少量参数（如分类头）

# Q50：BERT的Embedding层由哪三部分组成？各自的作用是什么？

- **Token Embeddings**：词嵌入，将输入token映射为向量，`[CLS]`标记用于分类任务
- **Segment Embeddings**：句子分段嵌入，区分句子A和句子B（用于NSP任务）
- **Position Embeddings**：位置编码，**通过学习获得**（非正弦/余弦固定编码），让模型感知词序信息

# Q51：为什么BERT选择使用可学习的位置编码，而不是Transformer原版的正弦位置编码？

BERT作者认为可学习的位置编码更灵活，能让模型在预训练过程中自主优化出最适合任务的位置表示方式，可能比固定数学公式更具表达力。代价是最大序列长度固定（如512），超出部分无法处理。

# Q52：BERT中的"双向"具体指什么？

"双向"指模型在处理每个词时能够同时关注其**左侧和右侧**的所有上下文信息。BERT通过MLM任务，让Transformer Encoder直接对每个词进行全向注意力计算，实现了真正的深度双向表示学习。

# Q53：BERT的两个预训练任务分别是什么？各自解决什么问题？

1. **MLM（Masked Language Model）**：解决**上下文感知**问题，通过遮盖15%的token让模型学习双向表示
2. **NSP（Next Sentence Prediction）**：解决**句子关系**理解问题，让模型判断两个句子是否连续

# Q54：MLM任务中，被选中的15% token分别如何处理？为什么要这样设计？

采用80%-10%-10%策略：

- **80%替换为[MASK]**：强制模型依赖上下文预测，学习双向表示
- **10%替换为随机词**：增强鲁棒性，防止模型过度依赖[MASK]标记
- **10%保持不变**：让模型学习"确认原词正确"的能力，确保所有token都能生成高质量上下文嵌入

**核心目的**：消除预训练（有[MASK]）与微调/推理（无[MASK]）之间的**分布不一致问题**。

# Q55：BERT的预训练和微调是什么关系？为什么采用这种两阶段策略？

- **关系**：预训练学习通用语言知识（基础），微调适配特定下游任务（应用）
- **优势**：实现**知识复用**。预训练成本高（需海量数据+算力），微调成本低（只需少量标注数据）。通过迁移学习，将通用语言特征高效迁移到数据稀缺的下游任务，显著提升效果。

# Q56：使用BERT处理长文本（超过512 token）有哪些常用策略？各适用于什么场景？

1. **Head-Only**：保留前510个token → 适用于关键信息在开头的新闻、摘要
2. **Tail-Only**：保留后510个token → 适用于结论在末尾的推理、对话记录
3. **Head+Tail**：头部+尾部拼接（如128+382或256+254）→ 适用于合同、法律文件等需要同时关注开头定义和结尾条款的场景

**注意**：Head+Tail会造成上下文不连续，不依赖中段论证的任务才适用。

# Q57：为什么BERT的中文模型以字为单位而不是词为单位？这会带来什么问题？

- **原因**：中文分词存在歧义，以字为单位可以避免分词错误传播，且字表规模可控（通常2-3万字），词表会过大
- **问题**：
    - 无法直接利用词级别的语义信息
    - 对生僻字只能以UNK代替，影响理解
    - 需要词向量的应用（如某些传统NLP任务）无法直接使用
