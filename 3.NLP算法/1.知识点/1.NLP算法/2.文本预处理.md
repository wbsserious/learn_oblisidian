**文本预处理**是指在自然语言处理（NLP）任务中，针对原始文本进行**清洗、转换、标准化**等一系列处理的过程。这是NLP工作流中至关重要的第一步，**旨在通过去除冗余和无关的部分，提高文本数据的质量和可用性**，以便后续的机器学习模型或深度学习模型能够更高效地学习和推理。
**核心目标**：
将**非结构化、含噪声、不规则**的自然语言文本，转化为**结构清晰、噪声可控、语义归一、符合模型输入规范**的标准化数据，为后续特征提取、语义表示与模型学习提供可靠基础。
**作用：**
	**增强文本表示**
	**提高数据质量**
	**降低计算复杂度**
	**改进模型性能**
# **主要目标**：
1.  **消除噪声干扰**
	- 原始文本通常包含标点符号、特殊字符、冗余空格、格式标记、无关符号等与**语义无关的干扰信息**，会增大模型学习难度并引入无效特征，预处理可有效**滤除噪声，提升数据信噪比**。
2. **实现文本归一化**
	- 自然语言存在大小写差异、时态变化、单复数、缩写、异体表达等形式差异，预处理通过统一化、词形还原、词根提取等操作，使**同一语义的不同表层形式被映射为统一表示**，避免模型将语义相同的文本判定为无关特征。
3. **降低数据稀疏性与计算复杂度**
	- 原始文本词汇空间庞大、冗余度高，通过分词、停用词过滤、低频词剔除等操作，可**压缩词汇表规模、降低特征维度**，减少模型参数量与计算开销，同时缓解数据稀疏问题，提升模型泛化能力。
4. **适配模型输入约束**
	- 主流 NLP 模型（如神经网络、统计学习模型）仅支持数值化、定长或结构化输入，无法直接处理原始字符序列。预处理通过分词、序列化、向量化等步骤，将自然语言转换为模型可接受的**离散符号序列或连续向量表示**，保证算法流程可执行。
5. **提升语义一致性与特征有效性**
	- 预处理可强化文本中核心语义单元的表示，弱化功能词与冗余信息的干扰，使模型更专注于**有效语义特征**的学习，从而提升文本分类、情感分析、命名实体识别、机器翻译等下游任务的精度与稳定性。
文本预处理是保障 NLP 模型**有效学习、稳定收敛、可靠泛化**的前提。
# 文本预处理流程（统计学NLP）
1. 原始文本获取与加载
2. 文本清洗（Noise Removal）
3. 文本归一化（Text Normalization）
4. 分词（Tokenization）
	将连续文本切分为具有语义意义的最小单元：
	英文：按空格、标点切分为单词
	中文：使用分词工具完成词语级切分
	目的：构建模型可处理的离散符号序列。
5. 停用词去除（Stop Words Removal）
6. 词形归一化（Morphological Normalization）
7. 文本向量化/序列化（Vectorization / Sequencing）
	将符号序列转换为模型可接受的数值形式：
		- 构建词汇表
		- 词索引映射
		- 生成固定长度序列
    目的：满足神经网络等模型的数值输入要求。
8. 长度标准化与填充（Padding & Truncation）
	对序列进行截断或填充，使其长度统一，满足批处理训练要求。
# 文本预处理流程（深度学习）
1. 文本处理
2. 轻量归一化
3. 子词切分（Subword Tokenization）

4. 添加模型特殊符号
5. 词表映射与数字化
	[[4.文本预处理—文本数字化]]
6. 序列长度统一

## **深度学习下的文本预处理流程着重点：**
- **子词切分 Tokenization（BPE/WordPiece）**
- **文本清洗（去噪）**
- **序列数字化、Padding / 截断**
- **特殊符号添加（[CLS]、[SEP] 等）**
# 统计学NLP算法和深度学习NLP算法的区别

| 对比维度      | 统计学 NLP（传统 NLP）             | 深度学习 NLP                          |
| :-------- | :-------------------------- | :-------------------------------- |
| **核心原理**  | 基于统计机器学习 + 人工规则             | 基于神经网络 + 表示学习                     |
| **特征来源**  | 人工设计特征（TF‑IDF、n‑gram、词性等）   | 自动从数据中学习特征                        |
| **文本表示**  | 稀疏高维表示（One‑hot、BoW、TF‑IDF）  | 稠密低维语义向量（Embedding）               |
| **上下文能力** | 弱，难以建模长距离依赖                 | 强，可捕获上下文与深层语义                     |
| **预处理依赖** | 依赖大量人工预处理（分词、去停用词、词形归一）     | 仅需轻量清洗、子词切分，人工规则少                 |
| **未登录词**  | 处理能力差，易失效                   | 泛化能力强，可处理新词、口语                    |
| **数据需求**  | 小数据即可运行                     | 需大规模数据，效果更明显                      |
| **算力要求**  | 低                           | 高（依赖 GPU/TPU）                     |
| **代表模型**  | Naive Bayes、SVM、HMM、CRF、LDA | CNN、RNN、LSTM、Transformer、BERT、GPT |
