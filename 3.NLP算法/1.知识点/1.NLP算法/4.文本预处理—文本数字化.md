**概念：** 是将经过预处理的文本符号序列，转换为**深度学习模型可计算的张量（Tensor）形式**的过程，是自然语言文本进入神经网络的**必经步骤**。
**作用**：
1. **核心作用**：将文本数据转化为大模型能够直接处理的数据。
2. 表达语义信息。
3. 提升模型性能。
#  向量化，词嵌入，张量化区分

| 名称       | 核心含义                     | 阶段位置           | 输出形式     | 与模型关系   |
| ---------- | ---------------------------- | ------------------ | ------------ | ------------ |
| **向量化** | 将文本转为**数值向量**的总称 | 文本表示阶段       | 一维向量     | 让语言可计算 |
| **词嵌入** | 学习到的**语义向量**         | 表示学习阶段       | 稠密低维向量 | 表达语义     |
| **张量化** | 向量转为**模型可输入的张量** | 模型输入前最后一步 | 多维张量     | 满足格式要求 |
- **向量化是大类**，包含词嵌入、One‑Hot、TF‑IDF 等。
- **词嵌入是向量化的一种高级方法**。
- **张量化是格式转换**，让向量能喂进深度学习框架。
数值化：文本数值化的作用是将文本符号转换为数值形式，使计算机能够理解、表示和计算自然语言。
向量化：**将文本映射为数值向量，实现文本的可计算化与结构化表示。**
词嵌入：**词嵌入将词语表示为稠密低维向量，能够编码语义信息，提升模型理解能力。**
张量化：**将向量转换为张量，满足深度学习模型的输入格式要求，支持批处理与高效计算。**
# One-Hot词向量表示
one-hot编码是一种将离散的分类变量转化为二进制向量的方法
```python
"cat" → [1, 0, 0]
"dog" → [0, 1, 0]
"fish" → [0, 0, 1]
```
**特点**：
1. 稀疏性。
2. 维度较高。
3. 信息丢失。
	无法必达词与词之间的关系。
**优点**：实现简单便于理解
**缺点**：
	1. 高维稀疏向量，维度灾难。
	2. 信息丢失，词语之间单独存在，无法体现出词语之间关系。
	3. 内存占用搞：向量长度过长，占用内存较高。
# word2Vec模型
**Word2Vec 是用于生成词嵌入的无监督模型，通过上下文学习语义，将词表示为稠密低维语义向量。分为 CBOW 和 Skip‑Gram 两种结构。**
核心思想：相似上下文的词，语义相似 → 向量也相近
**特点**：
1. 低维稠密向量，解决one_hot编码维度爆炸问题：基于one-Hot词向量进行训练，使用权重来作为词向量，高维词向量转换为低维稠密向量。
2. 保留语义，能够保留词语相似性
	1. 相似词彼此靠近。
	2. 支持语义计算。
3. 计算效率高
缺点：需要大量的语料来训练，且可能不适用于某些特定任务。
# 两种核心计算模式
## CBOW（Continuous Bag-of-Words）
- 用**上下文词**预测**中心词**
	- 训练快，
## Skip‑Gram
- 用**中心词**预测**上下文词**
- 语料大、效果好，对低频词友好**更常用**

| 对比维度 | CBOW                    | Skip‑Gram            |
|:-------- |:----------------------- |:-------------------- |
| 全称     | Continuous Bag‑of‑Words | Skip‑Gram            |
| 预测方向 | **上下文 → 中心词**     | **中心词 → 上下文**  |
| 训练速度 | **快**                  | 慢                   |
| 数据量   | 小数据也能用            | **大数据效果更好**   |
| 对低频词 | 效果较好                | **更擅长表示低频词** |
| 语料规模 | 适合中小规模语料        | 适合大规模语料       |
| 常见使用 | 简单、快速场景          | 通用、追求效果场景   |
# 词嵌入Word Embedding层
**Word Embedding定义：** 是将自然语言中的**词语**映射到**低维、稠密、连续的实数向量**的技术，是深度学习 NLP 中最基础的**文本表示方法**。
## 静态embedding
**定义**：静态 Embedding 是指**离散输入（如单词、ID）对应的嵌入向量在整个模型生命周期中具有全局唯一性**—— 同一输入无论出现在何种上下文（句子、位置、场景）中，其嵌入向量始终固定，仅随模型全局训练更新，不依赖局部上下文信息。
**核心特性**：
- **上下文无关性**：无法必达词与词之间的关系。
- **全局唯一性**：相同的单词在全局拥有相同的词向量。
- **训练特性**：嵌入矩阵的更新是全局的，可以跟随下游任务一同训练。
- **计算特性**：本质是查表操作，无上下文建模计算，时间复杂度为 O(1)（仅索引取值）。
**局限性**:
- 无法处理 “一词多义”
- **无上下文建模能力**
## 动态embedding
**定义**：动态 Embedding 是指**离散输入的嵌入向量由其全局嵌入矩阵和局部上下文共同决定**—— 同一输入在不同上下文（句子、位置、场景）中会生成不同的嵌入向量，向量值随上下文动态调整。
**核心特征**：
- **上下文依赖性**：同一输入的嵌入向量随其出现的语境、位置、相邻元素动态变化
- **局部适应性**：向量值反映输入在当前上下文下的具体语义
- **计算特性**：需结合上下文建模模块（如 Transformer 编码器），时间复杂度为 O(L⋅d2)（L 为上下文长度）
- **训练特性**：嵌入向量由 “全局嵌入矩阵 + 上下文模型参数” 共同决定，两者协同更新。
**典型实现与场景**：
- **实现方式**：BERT/GPT/Transformer 的输入层（`nn.Embedding` + 自注意力机制）、ELMo、GPT 系列模型；
- **适用场景**：复杂 NLP 任务（如机器翻译、问答系统、情感分析）、需处理多义性 / 上下文关联的场景。
**局限性**：
- **计算成本高**：上下文建模需多层神经网络运算，推理速度远低于静态 Embedding；
- **参数量大**：需额外维护上下文建模模块（如 Transformer 的注意力层、前馈层）的参数；
- **对数据量要求高**：需大规模语料训练才能学习到有效的上下文关联。
## 动静态embedding对比

| 维度       | 静态 Embedding                     | 动态 Embedding                   |
| -------- | -------------------------------- | ------------------------------ |
| 数学本质     | 全局固定的查表映射`E[i,:]`                | 全局查表 + 上下文建模的复合映射`f(E[i,:],C)` |
| 上下文依赖性   | 无（仅依赖输入本身）                       | 强（依赖局部上下文）                     |
| 向量唯一性    | 同一输入仅有一个向量                       | 同一输入可生成无限个上下文相关向量              |
| 一词多义处理能力 | 无                                | 强                              |
| 计算复杂度    | O(1)（查表）                         | O(L⋅d2)（上下文建模）                 |
| 参数量      | V×d（仅嵌入矩阵）                       | V×d+M（M 为上下文模块参数）              |
| 典型代表     | Word2Vec、GloVe、单独的`nn.Embedding` | BERT、GPT、Transformer 嵌入层       |
`E表示全局嵌入矩阵`
`C 表示输入 vi​ 的局部上下文（如相邻单词、句子结构、场景特征）`
`f 为非线性上下文建模函数（如自注意力机制）。`
# **Embedding 层（嵌入层）**
**定义**：Embedding 层（嵌入层）是深度学习中针对**离散类别型输入**（如单词索引、实体 ID、类别标签等）设计的可训练神经网络层，其核心是通过**线性映射**将高维稀疏的离散输入空间，投影到低维稠密的连续向量空间（嵌入空间），并在训练过程中学习输入的**语义 / 结构关联**，为后续神经网络模块**提供可计算**的连续特征表示。
**核心特点**：
1. **低维稠密**：词嵌入将每个词映射为低维稠密的向量，通常维度为50、100、200或300。
2. **捕捉语义和句法信息**：词嵌入能够捕捉词语之间的关系。
3. **迁移学习**：词嵌入能够在不同任务之间共享词向量，提高模型的泛化能力。
4. **适合需要padding的语句。**
**工作原理：**
Embedding 层本质上是一个**可训练的查找表（Lookup Table）**
**优点**： 
	 低维稠密变量，计算效率搞，占用内存小。
	 能够表达上下文含义。
	 适配神经网络，会和RNN，Linner层一同训练。
**缺点**：
	 需要足够的训练数据，否则容易过拟合。
	 无监督预训练的嵌入（如 Word2Vec）不一定适配特定任务。
	 对索引编码的依赖度高，预处理要求更严格
	 解释性差（黑盒特性）
