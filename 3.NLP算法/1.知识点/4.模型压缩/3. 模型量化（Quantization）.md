# 简介
**理论基础：浮点数的冗余**
标准神经网络训练使用 FP32（32位浮点数），每个权重占用4字节。但这一精度对推理而言往往是过度的，因为：
- 神经网络对噪声具有天然鲁棒性（冗余表达、激活函数的非线性性）
- 权重值通常集中在某个有界范围内，并非均匀分布于整个浮点数域
量化的数学目标是找一个映射 $Q: \mathbb{R} \to \mathcal{Z}$ ，将浮点值映射到离散整数集合，同时最小化量化误差。
**均匀量化（Uniform Quantization）** 的一般公式：
$$\begin{align}x_q &= \text{clip}\!\left(\left\lfloor \frac{x}{s} \right\rceil + z,\; 0,\; 2^b - 1\right)\\
\hat{x} &= s \cdot (x_q - z)\end{align}$$
其中$s$（scale）是缩放因子，$z$（zero-point）是零点偏移，$b$ 是目标位宽，$\hat{x}$ 是反量化还原值，$x - \hat{x}$  即为 **量化误差**（quantization error）。
**量化的目的**
1. 减少模型存储空间。
2. 降低运算内存，显存占用。
3. 加快计算速度
4. 降低功耗和成本
**常见低精度数据类型**
**FP16:**
- **半精度浮点数**（Half-Precision Floating-Point），它使用 16 位来表示一个浮点数。它的设计目标是在保持一定数值范围和精度的同时，显著减少内存占用和加速计算。
**BF16:**
- **Brain 浮点数**（Brain Floating-Point），也使用 16 位来表示一个浮点数，但其位分配方式与 FP16 不同。BF16 最初由 Google Brain 团队提出，旨在更好地模拟 FP32 的动态范围，从而在机器学习任务中表现更稳定。
**INT8:**
- **8位整数**（8-bit Integer），是一种**定点数**（fixed-point number）表示方法。与浮点数不同，整数没有指数和尾数位，它的值就是由其所有位直接表示的。INT8 主要用于**模型推理**阶段的极限量化，以实现最高的计算效率和最低的内存占用。
- **量化方式**：将原始的浮点数（通常是 FP32 或 FP16）映射到 INT8 范围。这个过程需要定义**缩放因子**（scale）和**零点**（zero-point）。
# 对称量化
这是最简单直接的一种方法。它的特点是**映射的区间总是关于`0`对称的。**
- **方法**：找到一批浮点数中**绝对值的最大值**，我们称之为`alpha`。那么我们的映射范围就是：`[-alpha, alpha]`。
- **优点**：计算简单，因为零点`Z`永远是`0`。

# 非对称量化
当我们的数据分布不均匀时，比如大部分是正数，只有少量负数，对称量化就会浪费掉一部分表达范围。非对称量化就是为了解决这个问题。
- **方法**：它不再要求关于 `0` 对称，而是直接找到数据的**最大值 `alpha`** 和**最小值 `beta`**，将 `[beta, alpha]` 这个范围完整地映射到整数范围（比如 `[-128, 127]`）。
- **优点**：能更紧密地“包裹”住原始数据，充分利用整数的表达空间，精度损失通常更小。
- **代价**：计算稍微复杂一点，因为需要引入**零点 `Z`**。

**对称 vs 非对称**的区别在于零点处理：对称量化令 z=0z=0 z=0，整数域关于零点对称，硬件实现更简单；非对称量化允许 z≠0z \ne 0 z=0，能更好地适应分布不均匀的激活值（例如经 ReLU 后全为正值的激活），精度更高但实现略复杂。

**均匀 vs 非均匀量化**：均匀量化的量化步长（step size）固定为 ss s，格点等距；非均匀量化允许格点不等距，理论上对非均匀分布的权重有更低的量化误差，典型方案是 k-means 聚类量化（将权重聚为 2b2^b 2b 类，用质心代替）和对数量化（对幂次分布更友好）。

# 实现路径
量化的实施时机分为**训练后量化（Post-Training Quantization, PTQ）** 和**量化感知训练（Quantization-Aware Training, QAT）** 两大路径：
![[Pasted image 20260326004858.png]]
## 训练后量化（Post-Training Quantization, PTQ）
- **描述：** 在模型已经完全训练好之后，再对其进行量化。这是一个独立于训练的过程。
- **优点：** 流程简单、快速，不需要重新训练，数据和计算成本低。
- **缺点：** 精度损失通常比QAT要大，尤其是在极低比特（如4bit）量化时。
- **子分类：**
    - **静态PTQ (Static PTQ):** 需要一小部分校准数据集来计算量化参数（如缩放因子和零点）。在推理时，激活值也使用这些预先计算好的参数进行量化，性能通常更好。
    - **动态PTQ (Dynamic PTQ):** 不需要校准数据。权重是离线量化好的，但激活值是在推理过程中“动态地”逐个计算其量化范围并进行量化的。这带来了额外的计算开销，通常用于RNN等动态性较强的模型。
- **PTQ 的关键细节：校准（Calibration）**。量化前需要用一小批"校准集"（通常数百到数千个样本）跑一遍前向推理，统计激活值的动态范围，从而确定每层的$s$ 和$z$。校准策略影响显著：MinMax 直接取最大/最小值，但对离群点敏感；百分位数（如第99.9百分位）更鲁棒；KL散度最小化（TensorRT的默认策略）则寻找使量化前后分布差异最小的截断阈值。
## 量化感知训练（Quantization-Aware Training, QAT）
**核心思想：伪量化 (Fake Quantization)** QAT 的精髓在于在训练的**前向传播**中，插入一个“伪量化”节点。这个节点的作用是模拟真实量化会带来的精度损失。它包含三个步骤：
- **量化 (Quantize)**：将输入的 FP32 权重或激活值，使用我们学过的公式 `round(S * x + Z)` 转换成 INT8 整数。
- **反量化 (Dequantize)**：立刻将上一步得到的 INT8 整数，用反向公式 `(q - Z) / S` 再转换回 FP32 浮点数。
- **结果**：经过这一来一回，我们得到的虽然还是 FP32 格式的数，但它的精度已经“劣化”了，它不再是原始的精确值，而是模拟了量化部署后才会出现的值。
- **QAT 的核心技巧：直通估计（Straight-Through Estimator, STE）**。量化操作 $\lfloor \cdot \rceil$的导数几乎处处为零，无法直接反向传播。STE 的解决方案是：前向传播使用量化后的离散值，反向传播时将量化算子的梯度视为恒等映射（即 $\partial x_q / \partial x \approx 1$ ），使梯度得以正常流通。这一"有用的近似"是 QAT 能够工作的数学基础。
# 主流精度格式及其权衡

|格式|位宽|存储（相对FP32）|典型场景|
|---|---|---|---|
|FP32|32 bit|1×|训练基准|
|BF16|16 bit|2× 压缩|训练/推理，大模型常用|
|FP16|16 bit|2× 压缩|推理，GPU原生支持|
|FP8（E4M3/E5M2）|8 bit|4× 压缩|训练加速（H100+）|
|INT8|8 bit|4× 压缩|推理主流，精度损失小|
|INT4|4 bit|8× 压缩|大模型推理，需GPTQ/AWQ等|
|INT2/INT3|2–3 bit|10–16× 压缩|极端压缩，精度损失显著|

FP8 近年受到高度关注，E4M3（4位指数3位尾数）适合权重，E5M2（5位指数2位尾数）动态范围更大适合梯度，两者结合用于训练混合精度（Transformer Engine, NVIDIA H100）。
