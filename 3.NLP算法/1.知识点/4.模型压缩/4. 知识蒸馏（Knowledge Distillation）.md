# 简介
核心思想是：一个训练充分的大模型（teacher）不仅"知道正确答案"，更在其输出的概率分布中编码了类别之间的结构关系——这是比硬标签（one-hot label）更丰富的监督信号，可以用来训练更小的模型（student）。
本质是：**让学生模型模仿老师模型的输出分布，而不仅仅是原始标签。**
**蒸馏 = 用大模型的“智慧”教小模型，让它“青出于蓝而胜于蓝”**。
跟剪枝、量化不同，蒸馏更侧重**知识迁移**，不是参数结构上的压缩。
# 蒸馏的三大知识来源
基础框架仅利用了 teacher 的输出层（logits），而更广义的蒸馏将知识分为三类：
**基于响应的蒸馏（Response-based）**：对齐 teacher 和 student 的最终输出分布，即上述经典框架。代表：Hinton KD, Born-Again Networks（BAN，用 student 作为新 teacher 继续蒸馏，可迭代多轮）。
**基于特征的蒸馏（Feature-based）**：将 teacher 中间层的特征表示也作为监督信号。代表性工作是 FitNets（Romero et al., 2015），让 student 的某一中间层去拟合 teacher 对应层的激活图：
$$\mathcal{L}_{feature} = \frac{1}{2}\left\| F_S(x) - r(F_T(x)) \right\|_2^2$$

其中$r(\cdot)$是一个可学习的回归层，用于处理 teacher 与 student 特征维度不匹配的问题。这类方法被大量应用于 BERT 压缩（DistilBERT、TinyBERT 均使用了中间层对齐）。
**基于关系的蒸馏（Relation-based）**：不对齐单个样本的表示，而是对齐样本之间的关系结构，例如角度关系、距离矩阵。代表：RKD（Relational Knowledge Distillation，Park et al., 2019），对 batch 内样本对之间的距离分布进行对齐：

$$\mathcal{L}_{RKD} = \sum_{(x_i, x_j)} \ell\!\left(d_T(f_T(x_i), f_T(x_j)),\; d_S(f_S(x_i), f_S(x_j))\right)$$
关系蒸馏在度量学习、自监督学习场景中尤其有效。
# 教师模型(Teacher Model)
> 一个**强大但笨重**的模型，已经“学富五车”，负责“教”学生。不参与学生模型训练过程的反向传播。
- **定义：** 复杂的、高性能的模型，通常是大型深度神经网络。
- **特点：** 参数量大，能够学习复杂的特征和关系。
- **作用：** 产生软目标（Soft Targets）作为知识。
# 学生模型(Student Model)
> 一个**小巧但聪明**的模型，正在“学习”，目标是“青出于蓝”。
- **定义：** 简化的、小型的模型。
- **特点：** 参数量较小，适用于资源受限的场景。
- **作用：** 学习真实标签的知识，同时模仿教师模型的输出分布。
# **蒸馏方式**
## 硬标签蒸馏
学生模型直接学习教师模型硬标签，即教师模型预测的具体类别作为学生的label。
损失函数公式：
$$L_{hard}=CE(y, p)$$
- $CE(y, p)$是交叉熵(Cross Entropy)
- $y$是真实标签
- $q$是Teacher在温度T下的softmax输出结果
## 软标签蒸馏
学生模型学习教师模型的硬标签和教师模型的软标签，将两种loss进行相加来更新学生模型的参数。
损失函数公式：
$$\begin{align}L_{total} &= (1-α)L_{hard} + α⋅L_{soft}\\L_{total} &= (1-α)CE(y,p) + α⋅CE(q,p)\end{align}$$
- $CE(y, p)$是交叉熵(Cross Entropy)
- $y$是真实标签
- $q$是Teacher在温度T下的softmax输出结果
- $p$是Student在温度T下的logsoftmax输出结果
- $CE(q,p)​$是KL散度，衡量两个概率分布之间差异的常用方法
- $α$是平滑系数，用于平衡硬损失和软损失的权重。通常取值在 0.5∼0.9 之间。
## 温度缩放
softmax-T公式
$$p_i=\frac{e^{(\frac{z_j}T)}}{\Sigma_je^{(\frac{z_j}T)}}$$
- $pi$是Student网络学习的对象，也就是所谓的软标签(soft targets)
- $zi$是神经网络 softmax 前的输出logits
- $T$是温度，最关键的参数。通常取值范围在2∼20之间。越大，输出分布越平滑，学生模型从错误类别中学到的信息越多。
	- 当 $T=1$：就是普通 softmax。
	- 当 $T→∞​$：所有 ，所以 ，输出趋近于 **均匀分布**。
	- 当 $T→0^+$：最大的  会被指数放大，其他项趋近于 0，输出趋近于 **one-hot 向量**（即硬决策）。
## 温度缩放的核心作用
温度 $T$ 是蒸馏中最关键的超参数。直观上，它控制软标签的"软化程度"：低温时分布尖锐（接近 one-hot），高温时分布平滑（类间相对关系更明显）。
- **T 越大 →$\frac{z_j}T$差距越小 → 指数后差距更小 → softmax 输出越接近均匀 → 分布越平滑**。
- 平滑的软标签能传递更多“类别间关系”的知识，是知识蒸馏成功的关键之一。
## 对比
从信息论角度看，硬标签的熵为 0（确定性最大），而软标签的熵 $H(p) = -\sum_i p_i \log p_i > 0$，包含了类间关系的额外信息。

# 模型架构
![[Pasted image 20260326014728.png]]
**总损失函数**（Hinton et al. 2015）：
$$L=α⋅T2⋅KL(σ(TzT​​)​σ(TzS​​))+(1−α)⋅CE(y,σ(zS​))$$
其中$z_T, z_S$分别是 teacher 和 student 的 logits，$\sigma$  为 softmax，$T$为温度，$\alpha \in [0,1]$  权衡两项损失。$T^2$ 系数是为了补偿高温时梯度幅值的缩小——温度 $T$ 使梯度缩小$1/T^2$ ，乘以 $T^2$ 后总梯度量级与$T=1$时保持一致。
# 蒸馏的变体谱系
![[Pasted image 20260326015743.png]]
**在线蒸馏**（Online Distillation）：teacher 并非预先训练好，而是与 student 同步训练。代表是 Deep Mutual Learning（DML，Zhang et al., 2018）——多个模型互相以对方的预测分布为软标签，实验表明集体学习效果优于单独训练，即便模型容量相同。
**自蒸馏**（Self-Distillation）：无需独立的 teacher，同一网络的更深层或后续训练阶段作为 teacher 监督更浅层。BYOT（Be Your Own Teacher）将深层特征投影后直接监督浅层，Born-Again Networks 将上一代 student 作为下一代的 teacher 迭代蒸馏多轮，每次均有性能提升。
**无数据蒸馏**（Data-Free KD）：当原始训练数据因隐私、版权无法获取时，用 teacher 生成或重建伪数据（通过反转 teacher 激活或 GAN 生成），再在伪数据上进行蒸馏。

# 知识蒸馏、剪枝、量化的对比与组合

|维度|知识蒸馏|剪枝|量化|
|---|---|---|---|
|压缩机制|训练更小的独立模型|移除原模型参数|降低参数精度|
|架构灵活性|极高（student 可完全不同架构）|受原架构约束|受原架构约束|
|是否需要训练|是（student 需完整训练）|通常需要微调|PTQ 无需，QAT 需要|
|精度损失|设计良好时可超越 teacher|低稀疏率时可忽略|INT8 通常 <1%|
|是否改变推理框架|否（student 是标准模型）|结构化剪枝不改变|需要整数运算支持|

三者并非互斥，标准的 LLM 部署压缩管线通常是：**蒸馏**（缩小架构规模）→ **剪枝**（去除冗余参数）→ **量化**（降低存储精度），叠加应用，在保持合理精度的同时实现 10–50× 的综合加速与压缩。
# 模型压缩和微调的关系
压缩程度越激进，微调的必要性越强。
## 量化
PTQ（训练后量化）通常**不需要**微调。INT8 精度损失极小，直接部署即可；INT4 在配合 GPTQ/AWQ 等二阶补偿算法时也基本不需要额外训练。但如果你用的是朴素 MinMax 量化且目标是 INT4 以下，误差会明显上升，这时用少量数据做 QAT 微调（几百到几千步）可以有效恢复精度。
## 剪枝
微调几乎**总是必要的**。剪枝会直接破坏网络已学到的权重协同关系，哪怕只移除 20% 的权重，不经微调的精度损失通常也是不可接受的。实践中的标准做法是：
- 低稀疏率（<50%）：短暂微调（几千步）通常足以完全恢复
- 高稀疏率（>70%）：需要更长的微调，甚至重新从头训练（彩票假设的验证实验表明，高稀疏子网络需要从随机初始化重训才能达到最优）
- 迭代剪枝本身就内嵌了"剪一点、训一轮"的循环，每一轮的"训"就是微调
## 知识蒸馏
蒸馏本身就是一个完整的训练过程，不存在"压缩后再微调"的问题。但有一个常见的后续步骤：如果 student 是在通用数据上蒸馏的，下游任务部署前通常还需要在任务数据上做有监督微调（SFT），这和蒸馏无关，是正常的迁移学习流程。
## 组合压缩管线中的微调策略
当三者叠加使用时，业界通常遵循这样的顺序：
蒸馏（得到小模型）→ 剪枝 + 微调 → 量化（PTQ 通常无需再微调）
一个重要的工程技巧是**一次性联合微调**（joint fine-tuning）：在剪枝后，把量化感知训练也同步加入微调过程，避免多次单独微调带来的累积误差和训练成本叠加。Meta 的 LLM.int8() 和 llama.cpp 中 GGUF 格式的模型基本不需要额外微调，就是因为它们的量化算法本身已经通过二阶优化补偿了精度损失，将微调的工作"内化"进了量化过程本身。
总结一句话：**量化可以不微调，剪枝通常必须微调，蒸馏本身就是训练**。
