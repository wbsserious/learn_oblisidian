# 模型简介
**基于 Transformer 编码器的双向表示
深度双向预训练语言模型**
历史地位：
**通用语言理解基座**，仅用 Transformer 编码器（无解码器），专注文本理解类任务。
**核心思想**：**双向编码**和**Transformer结构**
# 模型架构
BERT 完全由多层 Transformer 编码器堆叠而成，无解码器，结构统一用于预训练与微调。
## **输入表示（三嵌入求和）**
输入序列固定长度（最大 512），由三类嵌入相加得到最终输入向量：
- **Token Embedding**：词 / 子词嵌入，用 WordPiece 分词，含特殊标记
    - `[CLS]`：句首，用于分类任务的聚合表示
    - `[SEP]`：句分隔符，区分句对
    - `[MASK]`：掩码标记，用于 MLM 预训练
- **Segment Embedding**：区分句对（如 QA、NLI），`E_A`/`E_B`，单句则全为`E_A`
- **Position Embedding**：位置编码（可学习，非正弦），捕捉序列顺序
## **编码器层（重复堆叠）**
每层含**多头自注意力 + 前馈网络 + 残差 + 层归一化**：
- **多头自注意力（Multi-Head Attention）**
    - 每个头独立计算 Q/K/V，捕捉不同维度依赖
    - 双向：每个 token 可关注所有位置（无因果掩码）
    - 输出：`Attention(Q,K,V) = Softmax(QK^T/√d_k)V`
- **Add & Norm**：残差连接 + 层归一化，稳定训练
- **前馈网络（FFN）**：两层线性 + GELU 激活，`FFN(x)=max(0,xW1+b1)W2+b2`
- **Add & Norm**：再次残差 + 归一化
BERT模型双向注意力体现的方式：
**再多头注意力部分没有采用因果掩码，因果掩码会让注意力层只看到之前的内容。**
# 预训练机制
## 掩码语言模型（MLM）
- **目标**：预测被掩码的 token，强制模型学习双向上下文
- **操作**：随机掩码 15% 的 token，按比例处理：
    - 80% → `[MASK]`
    - 10% → 随机替换为其他 token
    - 10% → 保持原 token 不变
- **损失**：仅对掩码位置计算交叉熵损失
## 下一句预测（NSP）
- **目标**：判断句 B 是否为句 A 的后续，学习句间关系
- **操作**：输入`[CLS] A [SEP] B [SEP]`，用`[CLS]`输出做二分类
- **数据**：50% 为连续句对，50% 为随机句对
- **损失**：二分类交叉熵
- **目标**：判断句 B 是否为句 A 的后续，学习句间关系
- **操作**：输入`[CLS] A [SEP] B [SEP]`，用`[CLS]`输出做二分类
- **数据**：50% 为连续句对，50% 为随机句对
- **损失**：二分类交叉熵
# 微调范式（预训练→下游）
预训练完成后，**仅需添加任务特定层 + 小学习率微调**，即可适配各类 NLP 任务。
## 通用微调流程
1. 加载预训练 BERT 权重
2. 按任务构造输入（单句 / 句对）
3. 添加输出层（分类 / 标注 / 问答头）
4. 小学习率（2e-5~5e-5）、小批量（16/32）、少轮次（2~4 epoch）训练
## 典型任务适配
- **单句分类**（情感 / 意图）：取`[CLS]`输出→全连接→softmax
- **句对分类**（NLI / 相似度）：`[CLS] A [SEP] B [SEP]`→`[CLS]`→分类
- **序列标注**（NER / 分词）：每个 token 输出→标注层
- **问答（QA）**：输入`[CLS] 问题 [SEP] 文本 [SEP]`，预测答案起始 / 结束位置
