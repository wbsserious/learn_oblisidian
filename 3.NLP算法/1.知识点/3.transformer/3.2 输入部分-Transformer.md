输入部分主要包含两个部分：
**文本嵌入层**和**位置编码器**
$input_{embedding}$是通过常规**embedding**层，将每一个词的向量维度从$vocab_{size}$映射到$d_{model}$，由于是相加关系，自然而然地，这里的$positional_{encoding}$也是通过**Positional Encoding**层输出的一个$d_{model}$ 维度的向量。最终输出：
$$
input = input_{embedding}(x)+positional_{enconding}(x)
$$
# 文本嵌入层
**文本嵌入层**（Text Embedding Layer）是深度学习模型中**将文本（通常是单词或句子）转换为固定大小的向量表示的一个关键层。** 它的目标是将每个文本单元（如单词或子词）映射到一个高维空间中，以便模型能够更好地捕捉到词汇的语义信息和语法信息。
``` python
embedding = nn.Embedding(num_embeddings=10, embedding_dim=3, padding_idx=0)
# num_embeddings 词表大小
# embedding_dim 词维度
# padding_idx 填充词标号
```
# 位置编码器
**作用：**
Transformer 只有**自注意力**，没有循环、没有卷积，而自注意力具有**排列不变性（permutation-invariant）**，但语言是**有序**的，所以必须人为注入位置信息。
**定义：**
给序列中**每一个位置 pos**，生成一个 **d_model 维向量**，和**词嵌入逐元素相加**，作为模型真正的输入。
**非参数化位置编码方式**
## 位置编码器的核心数学性质
1. 每个位置的PE向量唯一。
2. **相对位置可以被模型学习。**
3. 可以外推到更长序列。
4. 维度=词维度=d_model。
# 两种主流位置编码
1. 固定正余弦编码。
2. 可学习位置编码。

# 固定正余弦编码
![[Pasted image 20260311174416.png]]
定义
**基于正弦（sin）和余弦（cos）函数，为序列中每个位置 `pos` 生成一个与词嵌入维度相同的固定向量，该向量不参与模型训练，仅通过数学公式预先计算，与词嵌入逐元素相加后输入模型。**
核心特征
**位置编码向量仅由位置 `pos` 和维度下标 `i` 决定，全程无可学习参数**。
公式：
$$\begin{align}
PE(pos,2i)&=sin(\frac{pos}{10000^{\frac{2i}{d_{model}}}})\\
PE(pos,2i+1)&=cos(\frac{pos}{10000^{\frac{2i}{d_{model}}}})
\end{align}$$
- $pos$：序列中token的位置。
- $i$：位置编码中的下表维度
- $2i$：偶数维度下标
- $2i+1$：奇数维度下标
- $d_{model}$：模型统一维度，一般和词维度一致。
- $10000$：频率基数（经验值）

| 符号          | 含义                   | 取值范围                      | 示例（d_model=512）            |
| ----------- | -------------------- | ------------------------- | -------------------------- |
| $pos$       | 序列中 token 的位置索引      | 0,1,2,...,seq_{len−1      | 第 1 个词：pos=0；第 10 个词：pos=9 |
| $i$         | 位置编码向量的维度下标          | 0,1,2,...,$d_{model}$​​−1 | i=0,1,...,255（共 256 个值）    |
| $2i$        | 偶数维度下标               | 0,2,4,...,$d_{model}$​−2  | 0,2,...,510                |
| $2i+1$      | 奇数维度下标               | 1,3,5,...,$d_{model}$​−1  | 1,3,...,511                |
| $d_{model}$ | 模型统一维度（词嵌入 / 位置编码维度） | 通常为 512、768、1024          | 512（原论文值）                  |
| $10000$     | 频率基数（原论文经验值）         | 固定常数                      | -                          |
|             |                      |                           |                            |
## 核心设计逻辑
1. 相对位置可以线性表达。
	利用三角函数的恒等变换：$$\begin{align}
	sin(pos+k)​&=sin(pos)cos(k)+cos(pos)sin(k)\\
	cos(pos+k)&=cos(pos)cos(k)−sin(pos)sin(k)
	\end{align}$$
	这意味着：**位置 pos+k 的编码向量，可由位置 pos 的编码向量通过线性变换得到**。
2. 无参数，计算高效。
3. 可无限外推长序列。
	训练时若模型只见过长度≤512 的序列，推理时遇到长度 = 1024 的序列，仍可直接通过公式计算出对应位置的编码，无需重新训练；而可学习位置编码无法处理训练时未见过的位置。
## 适用场景
- 原论文 Transformer 模型（机器翻译任务）；
- T5 模型（文本到文本生成）；
- 需要处理超长序列、且无法预先定义最大长度的场景（如长文档摘要）。
## 固定正余弦编码适合：
1. **机器翻译**（序列长度变化大）
2. **长文本 / 无限长度生成**
3. **训练数据较小**
4. **希望模型轻量化、无额外参数**
5. **需要跨长度泛化**

# 可学习位置编码
**思想**
可学习位置编码 = 位置也当成一个 “词” 来学嵌入
- 给序列中每个位置（0、1、2、3……）分配一个**可训练向量**
- 向量维度 = `d_model`（和词嵌入一样）
- 训练时和模型其他参数一起更新
- 前向时直接和词嵌入**相加**
**核心定义**
```python
位置编码矩阵 = nn.Embedding(max_seq_len, d_model)
```
- `max_seq_len`：模型能接受的**最大长度**（如 512、1024）
- `d_model`：模型维度
- 初始化：随机正态分布
- 训练：梯度下降更新。
## 可学习位置编码与固定正余弦编码对比表

| 项目     | 固定正余弦编码        | 可学习位置编码             |
| ----- | ------------- | ------------------ |
| 是否需要训练 | ❌ 不需要，固定       | ✅ 需要训练              |
| 来源     | 三角函数公式         | 随机初始化 + 学习          |
| 外推能力   | ✅ 支持任意长度       | ❌ 只能处理 ≤max_len 的序列 |
| 表达能力   | 较弱，固定模式        | ✅ 强，可拟合任务特有位置信息     |
| 主流模型   | Transformer、T5 | BERT、GPT、ViT        |
## 优势
1. 表达能力更强。
	模型可以根据任务学到**真正有用的位置信息**，而不是被固定函数限制。
2. 使用极其简单。
	就是一个普通 Embedding 层，代码好写、好改、好调试。
3. 效果在大多数任务上更好。
## 适用场景：
BERT、GPT、RoBERTa、ViT
## 可学习位置编码适合：
1. **预训练语言模型**（BERT/GPT 类）
2. **计算机视觉 Transformer**（ViT 系列）
3. **序列长度固定 / 可控**（分类、匹配、问答）
4. **数据量极大**，能学到复杂位置信息。
5. **追求强表达能力**，不在乎外推。
