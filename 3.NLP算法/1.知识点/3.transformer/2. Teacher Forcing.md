**Teacher Forcing** 是一种在训练序列到序列（Seq2Seq）模型时常用的技术，特别是在解码器（Decoder）的训练过程中。
[[5. 自回归语言建模]]
# 基础Teacher Forcing
**核心思想：** 
在**训练**时，使用**真实的目标序列（Ground Truth）** 作为解码器的输入，而不是使用解码器自己生成的输出。

**解决的核心问题：**
1. 误差积累。
2. 训练不稳定。
3. 收敛缓慢。

| 核心问题  | 具体表现                    | Teacher Forcing 解决效果        |
| ----- | ----------------------- | --------------------------- |
| 误差累积  | 早期预测错误会沿序列传递，最终生成结果偏离目标 | 输入始终为真实标签，彻底阻断错误传播，避免连锁反应   |
| 训练不稳定 | 梯度易消失 / 爆炸，长序列难捕捉依赖关系   | 稳定梯度传播，加速模型收敛，提升长序列学习能力     |
| 收敛缓慢  | 模型需从零学习正确序列依赖，训练效率低     | 直接提供正确上下文，快速学习序列模式，大幅缩短训练时间 |

**作用**：
- 能够在训练的时候矫正模型的预测，避免在序列生成的过程中误差进一步放大。
- teacher forcing能够极大的加快模型的收敛速度，令模型训练过程更快更平稳。、

**核心优点**：
1. 训练高效，收敛速度明显提升。
2. 训练时，稳定性强。
	在训练时，避免错误传递，尤其适合长序列任务，降低梯度消失 / 爆炸风险。
3. 易实现。

**核心缺点：
1. 训练与推理不匹配。
	训练时始终用真实标签，推理时只能用自身预测，模型从未学习 “错误后恢复”，易出现 “一步错、步步错” 的暴露偏差（Exposure Bias）
2. 泛化能力受限。
3. 长序列退化。
	训练时极度依赖真实标签，在推理时，长序列中的微小错误会被放大，影响整体推理效果。
**适用场景**

# 优化方案
Teacher Forcing的核心问题：**Exposure Bias（暴露偏差）**
**训练时使用真实数据，推理时使用自己生成的词与当作输入。**
## Scheduled Sampling（计划采样）
**思想**：
训练时，不是完全采用真实词，而是按照**概率混合**使用**真实词**和**模型生成词**。
	一开始：高概率用真实词（好收敛）。
	训练后期：慢慢提高模型自己生成词的概率。
	让模型提前适应错误输出。
**优点**：
- 简单，稳定，有效。
**缺点**：
- 训练缓慢
- 不是那么适配LLM模型
**适用场景**
- RNN / LSTM / GRU 序列生成
- 机器翻译、文本摘要、对话生成
- 希望**训练稳定 + 生成质量提升**的传统序列模型
**不适合**
- 超大规模 Transformer / LLM（成本太高，不常用）
## Curriculum Learning （课程学习）
**思想：**
先学短序列，再学长序列
	短序列：Teacher Forcing 快速学好
	长序列：降低 Teacher Forcing 比例，避免一步错全错
	让模型先学会简单的，再慢慢增加长度，减少长序列退化。
**优点：**
- 收敛更稳、效果更好
**缺点：**
- 需设计难度规则
## Fixed / Mixed Teacher Forcing（固定比例混合）
**思想：**
固定一个概率（如 50%）随机选择：用真实标签 or 模型输出。
**优点**
- 最简单、最快的改进
- 比纯 TF 更鲁棒
**缺点**
- 效果非常一般
## Sequence-Level Training（序列级训练）
**思想：**
不只预测下一个词，**优化整个序列的质量**。
**代表**
- MLE + RL
- PPO
- RLHF[[6. RLHF]]
- BLEU/ROUGE 作为奖励
**优点**
- 大模型对齐标配
- 真正解决“一步错步步错的问题”
- 大幅提高流畅度。
**缺点**
- 训练复杂，不稳定
- 超算资源需求大。
**应用场景**
- GPT 类大模型
- 对话系统、创作、摘要
- 需要人类偏好对齐的场景
## Non-Autoregressive Generation (NAR)非自回归生成
[[8. 非自回归生成]]
**思想**
**抛弃自回归**，一次性生成整句，没有时序依赖。
**优点**
- 推理极快
- **完全没有 Exposure Bias**
- 长序列不崩
**缺点
- 效果通常略差于自回归
- 训练更复杂
**应用场景**
- 机器翻译（最主流）
- 需要低延迟的线上场景
## Knowledge Distillation（知识蒸馏）
**思想**
用大模型的**软概率分布**监督小模型，不只学硬标签。
**优点**
- 小模型也能有大模型的生成能力
- 稳定、易训练
**缺点
- 需要先训一个大教师模型
- 不直接解决暴露偏差，只是间接改善
**应用场景
- 模型压缩
- 移动端 / 端侧部署
- 工业界落地
## Masked Prediction / Partial Teacher Forcing
**思想**
不是全用真实标签，也不是全不用，而是**部分token用真实标签**
**优点**
- 稳定 + 泛化平衡
- 适合 Transformer
**缺点**
- 效果中等
- 设计策略麻烦
**应用场景**
- 翻译、摘要
- 中等规模 Transformer
## Diffusion / Flow 模型（彻底替代方案）
[[9. Diffusion LLM：扩散式大语言模型]]
**思想**
通过**逐步去噪**生成序列，**完全不是自回归**。
**优点**
- 无 Teacher Forcing
- 无 Exposure Bias
- 长序列质量极高
**缺点**
- 推理慢
- 训练复杂
**应用场景**
- 高质量文本生成
- TTS 语音合成
- 未来新一代生成模型
## 选择策略
1. **Transformer / LLM 训练 = 纯 Teacher Forcing**
2. **想缓解暴露偏差：**
    - 传统模型 → **Scheduled Sampling**
    - 大模型 → **RLHF / PPO**
3. **想彻底消除暴露偏差：**
    - **Non-Autoregressive**
    - 或 **Diffusion**

# 当前业界主流范式
[[7. 2026年业界主流训练范式]]
**预训练阶段纯 Teacher Forcing + 对齐阶段序列级强化学习（RLHF/PPO）+ 全流程隐式 Curriculum Learning**
